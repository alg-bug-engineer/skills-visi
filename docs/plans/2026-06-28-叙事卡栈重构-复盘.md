# 领导演示 · 项目整体复盘（叙事 / 播报 / 地图一致性 + 故事性评估）

> 日期：2026-06-28
> 视角：以"技术系统向非技术领导演示"为目标，复盘文案、语音播报、地图动作是否对应一致，
> 以及领导能否感受到①智能体带来的改变 ②对一线民警经验的吸收复用 ③一个完整、有故事性的剧本。
> 关联：本次 1-3 项开发（右上角叙事卡栈 / 镜头连贯 / 问题验证-治理建议层级）已落地。

## 1. 剧本主线（三条路径）

| 路径 | 输入 | 故事承诺 |
|------|------|----------|
| 单点·首诊 | 「奥体西路与经十路交叉口，晚高峰南北向经常拥堵，垂直方向不能溢出」 | 听懂口语 → 看清结构 → 数据验证 → 给出建议 → 经验入库 |
| 单点·复用 | 同句（已有 skill） | 「锁定路口」即带出历史约束，体现经验不浪费 |
| 干线 | 「奥体西晚高峰哪些路口比较堵？」→ 点选 | 一条路扫描发现堵点 → 下钻单点 |

8 步（`ANALYSIS_STEP_LABELS`）：理解描述 / 锁定路口 / 路口结构 / 运行数据 / 问题印证 / 原因诊断 / 治理建议 / 经验固化。

## 2. 文案 ↔ 播报 ↔ 地图动作 对应表

| 步 | 右栏文案(首行结论) | 语音(voice_narration) | 地图动作 | 新增·右上角叙事卡 |
|----|--------------------|------------------------|----------|--------------------|
| 理解描述 | 复述民警原话+结构化 | `guide.understand` | 不动 | — |
| 锁定路口 | 「已匹配/定位…路口」(+历史约束) | `templates.intersection`/`skillReuseHint` | 连续下钻(不回跳) | 认知头出现 |
| 路口结构 | 轴路名 + 关注/保护方向 | `cognitionWithRoads`/`axisRoads` | 渠化淡入 + 轴路名 | 认知头：臂·道/zoom/层级/方向角色 |
| 运行数据 | 「整体饱和度 X，南北偏高」 | `saturation`/`imbalance`/`directionFocus` | HUD 错峰 | 运行数据逐项✓追加 |
| 问题印证 | 「近7日5天超标，与描述一致」 | `evidenceChronic`/`evidenceDow` | 证据高亮 | 问题验证段→自动折叠 |
| 原因诊断 | 规则结论一句 | `ruleIntro`/`ruleHit` | 配时环/绿波小窗 | (运行数据·干线行) |
| 治理建议 | 「南北加绿12s，已护东西不溢出」 | `suggestion`/`suggestionConfirm` | 建议箭头 | 治理建议段（折叠条目下） |
| 经验固化 | 吸收 trace + 写技能包 | `absorption.*`/`skillBuild*` | dim 不可 pan | — |

**一致性结论**：三条通道（看 / 听 / 图）在每一步均有对应锚点，无"只播不显""只显不播"的断裂。本次改造把原先散落四角的 证据卡(左上)、建议卡(右上)、运行数据(右下) 收敛为右上角一列，使"看"的通道从多焦点变为**单列时间线**，与"听"的线性播报、"图"的单步焦点三线合一。

## 3. 领导可感知的三个价值点

### ① 智能体带来的改变 —— 强
- 理解过程逐行打字 + 语音同步 + 地图随推理变化，呈现"**智能体在干活**"而非有人报幕（无"第N幕/演示模式"字样，符合 C2）。
- 本次右上角叙事卡"**随叙事向下生长**"，把抽象推理变成可见的"结论在累加"，领导一眼看到系统在**逐项坐实**判断。

### ② 一线经验的吸收复用 —— 中强（仍是全剧最易被低估的一环）
- 复用路径在「锁定路口」首行直接带出历史约束（「垂直方向不能溢出」纳入本次方案），经验不再埋在长文中。
- 固化阶段保留 L3 交错写技能包 + 吸收 trace，闭环完整。
- **风险**：经验价值集中在"锁定路口"一行与最后固化两处，中段（运行数据/诊断）领导可能淡忘"这是民警约束在起作用"。**建议**：治理建议段在叙事卡中对受约束项加一枚标记（如「↩ 源自民警约束」），让经验在"产出建议"时再被点亮一次。

### ③ 可靠 / 可推广 —— 强
- 运行数据逐项✓ + 问题验证（常发/投诉/饱和度）构成"证据链"，回答"靠不靠谱"。
- 技能固化 + 复用路径回答"能不能推广"。

## 4. 故事完整性 —— 断点与本次修复

| 原断点（重构前） | 现状 |
|------------------|------|
| 镜头每步从市域重新下钻、闪烁割裂 | **已修**：单调 zoom + userInteracted 保留视角，只前进不回跳 |
| 运行数据(右下)、证据(左上)、建议(右上)三处分散，视线乱跳 | **已修**：收敛为右上角单列时间线 |
| 问题验证与治理建议同级平铺，层级不清 | **已修**：问题验证出现后自动折叠，治理建议在其条目下方，呈"印证→对策"层级 |
| 认知信息（结构/缩放/层级）无承载，下钻提示缺失 | **已修**：认知头承载身份/臂道/实时zoom/层级/方向角色/下钻提示 |

**仍存的故事性提升空间（建议后续）**：
1. 经验在"治理建议"处二次点亮（见 §3②）。
2. 干线→单点的"选型瞬间"可加一句过场语音（当前主要靠地图高亮），强化"一条路问下来、聚焦到这个口"的因果。
3. 固化完成后，可在右上角叙事卡尾部追加一行「已写入技能包，同类路口将自动复用」，让"留存"在主视线收尾，而不仅在左抽屉。

## 5. 演示动线建议（讲解员，不进 UI）

约 7–9 分钟：单点首诊(0:00–4:00) → 确认建议+固化(4:00–5:00) → 同句复用看历史约束(5:00–6:00) → 干线扫描选型接单点(6:00+)。
关键停顿点（空格）：「路口结构」后、「问题印证」后——此时右上角叙事卡刚长出一段，最适合讲解。

## 6. 验收口径

- 看/听/图三线在 8 步均有锚点（§2）。
- 全程无"演示/报幕/第N幕"字样（C2）。
- 功能 100% 保留：8 步、语音、配时环、绿波、固化、抽屉、空格暂停（C3）。
- 镜头只前进不回跳；手动拖动后不被拉回。
- 右上角单列：认知→运行数据→问题验证(折叠)→治理建议。

> 一句话评价：重构后，领导看到的是**一份在眼前逐行写出来的诊断书**——系统听懂民警的话、把数据一条条核实、给出带约束的对策、并把这套经验存起来下次自动复用。剩余打磨集中在"让民警经验在产出建议时再被看见一次"。
