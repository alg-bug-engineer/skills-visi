## P4 集成

### 任务

1. `orchestrator._diagnose`：在 `rule_engine` 之后调用 governance service，写入 `data_payload["flow_timing_governance"]`；

### 文件

- `backend/intersection_agent/services/orchestrator.py`

## P5 ECS 部署（原生方式，非 Docker）

### 任务

1. `scripts/prod-start.sh` — 构建 frontend-v2、启动 uvicorn `0.0.0.0:8011`、配置 Nginx；
2. `scripts/prod-stop.sh` — 停止生产后端；
3. `deploy/nginx.host.conf` — 宿主机 Nginx 静态资源 + `/api` `/health` 反代 `127.0.0.1:8011`；
4. `deploy/intersection-agent-backend.service` — systemd 单元（可选）；
5. `deploy/README.md` — ECS 部署指南；
6. `config.py` 增加 `cors_origins`；
7. `scripts/dev-v2.sh` / `vite.config.ts` 支持 `BIND_HOST` 环境变量；
8. `docker-compose.prod.yml` — 保留作 Docker **备选**，非默认路径。

### 环境变量（生产）

| 变量 | 说明 |
|------|------|
| `HOST` | `0.0.0.0` |
| `PORT` | `8011` |
| `CORS_ORIGINS` | `*` 或具体域名 |
| `PGHOST` 等 | 数据库连接 |
| `DASHSCOPE_API_KEY` | LLM |

前端构建：`VITE_API_BASE=`（空，同源反代）。
