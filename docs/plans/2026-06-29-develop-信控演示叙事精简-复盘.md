> 日期：2026-06-29 · 分支：`develop` · 状态：**首个可用版本（MVP）**
> 前置基线：`main` @ `267aaa2`（供需匹配度主轴 + 流量溯源接入）
> **禁止合并 `main`**：本分支为演示叙事精简试验线，与 `main` 功能取舍不同。

---

## 1. 分支定位

在 v3.1「供需匹配度主轴」基础上，将领导演示叙事收敛为：

- **左侧**：只展示**运行数据与证据**，不放治理结论
- **右侧 / 语音**：保留治理建议、供需匹配度 headline
- **删减**：流量溯源、干线绿波、群众投诉、整体饱和度、规则结论行等非主轴内容

目标：演示时「左看数、右看建议」，避免左右重复结论、减少语音堆叠。

## 3. 左侧面板原则（数据 / 证据，无结论）

### 3.1 已删除的左侧条目

- 真实渠化数据、层级/zoom、操作提示
- 群众投诉、路臂表达、转向极差、左转分析
- 日计划时段、节点位置、整体饱和度
- 流量溯源（播报、文案、地图）
- 规则结论行（如「过饱和需增加绿灯」）
- 干线绿波及相关工具栏、小窗
- **信号调整秒数**（如「信号调整 增加 0 秒」）、绿信比等配时结论

### 3.2 保留 / 新增的运行数据

| 类别 | 示例 |
|------|------|
| 基础指标 | 延误指数、失衡系数（方向失衡） |
| 常发拥堵 | 近 N 天偏堵天数 |
| **转向级（与治理建议一致）** | 东左转饱和度 1.77、东左转绿灯利用 177%、北左转绿灯利用 26% |

转向指标来源：`flowTimingGovernance.primary_diagnosis.turn_balance`（`over` / `spare`），由 `buildNarrativeRuntimeItems` 注入左侧面板。

### 3.3 结论过滤机制

`shouldSkipRuntimeMetric(label, value)`：

- 标签黑名单：`信号调整`、`绿信比`、`规则结论`、`证据链` 等
- 值模式：`增加 0 秒`、`+3 秒` 等配时结论

应用于 `usePresentation.setHud` 与 `buildNarrativeRuntimeItems` 两处，保证 HUD 缓冲与运行数据面板一致。
