## 1. 设计目标

| 目标 | 说明 |
|------|------|
| **像在用系统** | 无报幕、无演示开关、无「第 N 幕」 |
| **一屏一焦点** | 同一时刻主舞台突出 1 个结论 + 1 个空间结构（渠化/地图） |
| **能力全保留** | 8 步、语音、固化、配时环、绿波、指标条等均在，按步骤 **依次出现** |
| **故事靠步骤** | 右栏仍是「理解过程」时间线，靠步骤推进讲清价值 |

---

## 2. 整体布局（与现网一致，微调秩序）

```
┌────────────────────────────────────────────┬──────────────────────────┐
│ 路口 GIS          [静音] [配时环图] [干线绿波] │  ANALYSIS · 理解过程      │
├────────────────────────────────────────────┤  智能体推理        [收起] │
│                                            │                          │
│         地图 + 渠化（当前步骤主焦点）         │  ✓ 理解描述               │
│                                            │  ✓ 锁定路口               │
│   [本步仅突出 1～2 个 overlay 组件]          │  ● 路口结构               │
│                                            │    经十路与奥体西路…       │
│                                            │    [查看详情 ▾]           │
├────────────────────────────────────────────┤                          │
│  （MetricStrip 在本步后半段才 slide-up）     │                          │
└────────────────────────────────────────────┴──────────────────────────┘
│  输入区（分析中可置底只读）                                                  │
└──────────────────────────────────────────────────────────────────────────┘
```

**与 v1 方案差异**：

| 原方案 | v2 确认 |
|--------|---------|
| 顶部 5 幕进度条 | **取消** |
| 演示/专业模式切换 | **取消** |
| 隐藏 MetricStrip / 配时环 | **保留**，改为步骤内错峰揭示 |
| 幕标题「第三幕 · 数据验症」 | **取消**，用步骤名「问题印证」 |

## 3. 视觉规范

### 3.1 色彩（不变）

| 语义 | 色值 |
|------|------|
| 关注方向 | `#ff6b4a` |
| 保护方向 | `#6dffb5` |
| 理解过程-进行中 | 现有青蓝 |
| 理解过程-完成 | 绿色勾 |

### 3.2 文案层级（右栏每一步）

```
步骤标签（如「路口结构」）          ← 改业务化，仍 8 步
首行结论（step_summary，≤40 字）   ← 新增，默认可见
[查看详情 ▾]                      ← 展开后：link 列表、Skill ID、规则名等
```

**禁止作为主视觉**：报幕、演示、样例、第 N 幕。

### 3.3 动效

| 动作 | 说明 |
|------|------|
| 步骤切换 | 理解过程下一步展开，上一步自动折叠（现有逻辑） |
| 组件 reveal | MetricStrip / 证据卡 / 小窗：slide-up 350ms，**替代** 一进来就全显示 |
| fly_to | S3 路口结构步，800ms |

## 4. 分步骤交互（单点）

### 4.1 理解描述（STEP 0）

| 元素 | 行为 |
|------|------|
| 地图 | 市域或上一会话状态，无分析高亮 |
| 右栏 | 复述用户原话；首行：「已理解拥堵描述与时段约束。」 |
| 语音 | `guide.understand`（文案改业务口吻） |
| 输入 | 发送后置底只读 |

### 4.2 锁定路口（STEP 1）

| 元素 | 行为 |
|------|------|
| 地图 | 可选轻微定位；无渠化 |
| 右栏首行 | 「已匹配奥体西路与经十路路口。」 |
| **Skill 复用** | 首行追加：「载入历史约束：垂直方向不能溢出。」Skill ID 在「查看详情」 |
| 语音 | `templates.intersection` + 可选 `skillReuseHint` |

**禁止**：顶部黄色「已加载历史经验」报幕横幅（改为步骤内自然叙述）。

### 4.3 路口结构（STEP 2）

**地图动作**（locate + links + channelization 合并呈现，SSE 仍逐步到达）：

1. fly_to zoom 17.8
2. 3D 渠化淡入，底图 dim 0.85
3. 路臂标签：东/西/南/北 + 路名
4. 关注/保护着色 + 简化图例

| 元素 | 行为 |
|------|------|
| 右栏首行 | 「东西向为经十路，南北向为奥体西路；南北向为关注方向。」 |
| link 10 条明细 | **查看详情** 内 |
| MetricStrip | **尚未出现** |
| 语音 | `cognitionWithRoads` / `axisRoads` |

### 4.4 运行数据（STEP 3）

子 phase 推进时 **HUD 错峰**（能力不删）：

| 子阶段 | 地图主焦点 | 暂不出现 |
|--------|------------|----------|
| traffic / direction | 关注方向 pulse + 1 张饱和度 HUD | MetricStrip、失衡横幅 |
| saturation | 更新 HUD | 配时环小窗 |
| imbalance | 失衡 HUD **或** 横幅（二选一主显） | 另一个延后 1 beat |
| granularity / timing / corridor | 转向/配时/干线 HUD **择要** | InsightStack |

**本步末尾**：MetricStrip slide-up（四向指标 **保留**）。

| 右栏首行 | 「晚高峰整体饱和度 0.92，南北向偏高。」 |
| 语音 | directionFocus + saturation（摘要） |

### 4.5 问题印证（STEP 4）

| 元素 | 行为 |
|------|------|
| 证据 note | slide-in（≤3 行摘要） |
| InsightStack | 与证据 note **同帧或紧随其后** |
| MetricStrip | 已可见，保持 |
| 右栏首行 | 「近 7 日 5 天超标，与「经常拥堵」一致。」 |
| 语音 | `evidenceChronic` 等 |

### 4.6 原因诊断（STEP 5）

| 元素 | 行为 |
|------|------|
| 配时环 / 绿波 | **首次**自动弹出小窗（若数据有）；工具栏按钮此前已可见可点 |
| 右栏首行 | 规则结论一句（如「配时相对流量偏紧。」） |
| 语音 | `ruleIntro` + `ruleHit` |

### 4.7 治理建议（STEP 6）

| 元素 | 行为 |
|------|------|
| 建议 note | 揭示：方向 + 秒数 + 约束说明 |
| 确认 | 现有 confirm 流 **保留** |
| 右栏首行 | 「建议南北向加绿 12 秒，已考虑东西向不溢出。」 |
| 语音 | `suggestion` / confirm 引导 |

### 4.8 经验固化（STEP 7）

| 元素 | 行为 |
|------|------|
| 右栏 | eyebrow 仍为 `ABSORPTION` / 理解过程「经验固化」步；**不改**「第五幕」 |
| 吸收 trace | 保留自言自语体；首行加一句业务摘要（非报幕） |
| 左抽屉 | L3 交错写文件 **完整保留** |
| 地图 | dim，吸收期间不可 pan |
| 语音 | absorption.* + skillBuild* **全流程保留** |

## 5. 干线路径 UI

### 5.1 理解 + 扫描

| 阶段 | 主视觉 |
|------|--------|
| 用户发问 | 理解过程打字；地图可显示道路粗定位 |
| 扫描完成 | 沿路 polyline + **CorridorScanSidebar**（保留） |

### 5.2 侧栏行（精简展示，不删功能）

每行默认显示：

```
排名 · 路口名 · 饱和度
```

展开行可看：投诉、延误等原有字段（点击行或「详情」）。

### 5.3 选型 → 单点

选中路口：侧栏高亮 → fly_to → 进入 **4.3 路口结构** 起单点 8 步。

## 8. 彩排时间轴（讲解员参考，不进 UI）

约 **7–9 分钟**（单点 + 固化 + 复用 + 干线节选）：

| 时段 | 操作 |
|------|------|
| 0:00 | 口头背景；输入单点句 |
| 0:30–4:00 | 跟随理解过程 8 步；空格在「路口结构」「问题印证」后可选停 |
| 4:00 | 确认建议 → 确认固化 → 看吸收/写包 |
| 5:00 | 新对话，同句 → 看「锁定路口」历史约束 |
| 6:00 | 输入干线句 → 扫描 → 选型 → 走几步单点 |

## 9. UI 验收

| ID | 操作 | 期望 |
|----|------|------|
| UI-NO-DEMO-STR | 全路径走查 | 无「演示、报幕、第 N 幕、样例模式」字样 |
| UI-FEAT-ALL | 功能清单 | 8 步、语音、MetricStrip、配时环、绿波、固化均触达 |
| UI-FOCUS-S3 | 路口结构步 | 无 MetricStrip；渠化 + 图例为主 |
| UI-FOCUS-S4 | 运行数据步 | MetricStrip 在本步内出现，非一开始全屏 |
| UI-SKILL-INLINE | 复用 | 历史约束在「锁定路口」步首行，非顶栏横幅 |
| UI-FOLD | 查看详情 | link / Skill ID 可展开 |
| UI-CORRIDOR | 干线 | 侧栏 + 地图 polyline + 选型进单点 |
| UI-PAUSE | 空格 | toast 无幕编号 |
