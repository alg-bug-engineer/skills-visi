# 演示路口嗅探 · 阈值统一 · 检查单对齐开发计划

> **版本**：2026-06-27  
> **目标**：支撑市委书记汇报演示——故事落脚在「信控配时与流量不匹配」，诊断聚焦 **饱和度 / 失衡 / 空放 / 溢出** 四维；数据以 **完整性与真实性** 优先，时间窗口可锚定演示日。  
> **关联文档**：[路口场景认知与问题诊断检查单.md](../路口场景认知与问题诊断检查单.md)、[2026-06-26-四类问题与ECS部署开发计划.md](./2026-06-26-四类问题与ECS部署开发计划.md)

## 1. 背景与原则

| 原则 | 说明 |
|------|------|
| **不做全量检查单对齐** | 37 项场景认知 + 16 项诊断全部接入成本高、演示易碎；仅对齐与四维演示及流量-配时叙事相关的子集 |
| **阈值以检查单为真源** | `docs/intersection/common/thresholds.yaml` 为权威；backend 与之统一 |
| **演示日优先** | DWD 可用区间 `2026-06-08 ~ 2026-06-16`；DWS 按 `day_of_week` 周模式；汇报可固定「周五晚高峰」叙事 |
| **故事主线** | 一线经验（检查单/技能）→ 数据印证 → 四维问题 → 流量-配时失配 → Skill 沉淀 |

## 2. 数据库嗅探结论

### 2.1 嗅探方法

- 脚本：`backend/scripts/probe_schema_and_demo.py`（完备度打分）+ 扩展 SQL（四维指标）
- 时段：**晚高峰 17:00–19:00**（`step_index` 204–228）
- 四维判定阈值（与检查单一致）：
  - **饱和度 S**：`saturation_max ≥ 0.80`
  - **失衡 I**：`imbalance_index ≥ 0.30` 或 `turn_saturation_spread ≥ 0.60`
  - **空放 E**：`green_utilization_min < 0.60` 或 `green_utilization_avg < 0.60`
  - **溢出 B**：`queue_storage_ratio ≥ 0.80` 或 `queue_max_m ≥ 100`

### 2.2 数据环境摘要

| 项目 | 结果 |
|------|------|
| DWD 明细日期 | 2026-06-08 ~ 2026-06-16，474 路口 |
| DWS 晚高峰覆盖最佳星期 | **周六 dow=6**（121 路口）；**周五 dow=5** 次之，更适合「工作日晚高峰」叙事 |
| 推荐演示锚定日 | **2026-06-13（周五）** 晚高峰 17:00–19:00 |
| 完备度满分路口（9/9 旗标） | 二环东路与工业南路、奥体中路与工业南路、崇华路与新泺大街 |

> **数据注意**：部分路口 `saturation_max` 原始值 > 1.2（如崇华路新泺大街、奥体中路新泺大街），与检查单 `calculate_saturation` 口径 `min(1.5, V/C)` 不一致。演示展示与规则判定应 **统一 cap 到 1.5**，叙事用「明显过饱和/高饱和」而非裸值。

### 3.1 真源与加载路径

| 现状 | 目标 |

**推荐做法**（二选一，P1 任务内定）：

### 4.2 问题诊断（四维 + 流量-配时）

| item_id | 问题码 | 四维 | 检查单要点 | backend 现状 | P 级 |
|---------|--------|------|-----------|-------------|------|
| — | — | 饱和度 | `saturation.high` / `oversaturation` | `rule_oversaturation` 等 | **P1** 阈值 |
| `service_imbalance` | `service_imbalance` | 失衡 | imb≥0.30 **或** 极差≥0.60，**连续15min** | 瞬时判定；极差阈 0.25 | **P2** |
| `empty_green` | `empty_green` | 空放 | 利用率<0.60，**连续15min** | 阈 0.50，无持续 | **P2** |
| `spillback` | `spillback` | 溢出 | risk≥0.80 或 qsr≥0.80 | `rule_spillback_overflow` 用停车+代理 | **P2** |
| — | — | 流量-配时 | Spearman τ + 绿信比 | `flow_green_fit` + `rule_flow_green_mismatch` | **P1** 叙事增强 |
| `cycle_timing` | `cycle_timing_issue` | 配时 | >190s 或 <60s | 已有 | — |
| `plan_granularity` | `plan_granularity` | 配时 | 方案数≤5 | 阈 6 | **P1** |
| `public_complaint` | `public_complaint` | 体验 | 任一非空即触发 | 需投诉≥5 且高饱和 | **P3** 演示模式放宽 |

### 4.3 四维与规则/服务映射（目标态）

```
饱和度  ← inter_evaluation.saturation_max + turn_saturation
         ← rule_oversaturation, flow_timing_governance.problems[saturation]

失衡    ← inter_evaluation.imbalance_index + turn_saturation spread
         ← rule_service_imbalance + rule_turn_imbalance（极差阈 0.60）
         ← 新增：5min 时序连续 15min 窗口（复用 diagnosis_imbalance_logic）

空放    ← green_utilization 时序
         ← rule_empty_green（阈 0.60）+ 15min 持续（复用 diagnosis_empty_green_logic）

溢出    ← turn_perf queue_storage_ratio / spillback_risk
         ← rule_spillback_overflow + flow_timing_governance.problems[spillback]

流量-配时失配 ← timing_profile.flow_green_fit + min_green 赤字
              ← rule_flow_green_mismatch（叙事主轴）
```
