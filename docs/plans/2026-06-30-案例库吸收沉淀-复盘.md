- 后端 `CaseLibraryService.list_all()` 完整解析 `expert_knowledge.md`：补回此前丢弃的「出现频次 / 典型表现 / 方案频次 / 代表案例 #N（标题+摘要）」。
- 新路由 `GET /api/v1/cases/industry?q=`（行业案例，含关键词过滤）、`GET /api/v1/cases/intersections`（路口案例，由 `IntersectionProfileStore.load_all()` 聚合，**仅保留已形成治理方案 solution_ref≠∅** 的路口）。
- 前端 `CaseLibraryPanel.vue`：「理解过程 / 经验库」旁第三 Tab「案例库」，内含 行业/路口 两子 Tab；行业＝场景列表+搜索+展开问题/方案/代表案例，路口＝结构化卡（场景认知→诊断成因→治理方案与成效）。

### 需求 2 · 验证浮层 Toast（替代经验吸收卡片）
- `store.add_cognition/add_diagnosis/add_solution_ref` 返回去重结果 `inserted/exists/updated`，并随 SSE `experience_cognition/diagnosis/solution` 事件下发 `action`。
- 前端 `useAbsorptionToasts.ts` + `ExperienceAbsorptionToast.vue`：右下角 slide-up 浮层。
  - 认知：验证后「已验证…」/「未找到数据支撑，已记录」+ 标签化理解 + 已插入/已存在/已更新认知经验库。
  - 诊断：「已吸收…并入库」+ 结构化标签；命中去重→「该诊断经验已存在（不再插入/更新）」。
- **删除** `ExperienceAbsorptionPanel.vue` 可视卡片（保留 `useExperienceAbsorption` 状态机供语音/暂停门控），避免重叠重复。

### 需求 3 · 治理建议深度融合 + 可溯源闭环
- `SuggestionResult.references`：由 `case_matches`（专家场景）+ `reused_experience`（本路口经验）确定性派生，随 `suggestion` 下发。
- `NARRATIVE_PROMPT` 新增原则 9：必须深度融合「用户经验 + 同类案例 + 上游溯源」三源，并让建议体现「依据来源」。
- 前端建议卡渲染「参考依据」chip：hover 显示概要+来源类型+ID，点击 → 切到案例库并定位（`industry:<scenario_id>` / `intersection:<inter_id>`，`caseReference.ts` 解析锚点）。

## 三、手验清单

1. 顶部「案例库」Tab：行业案例可搜索（如「学校」「绿波」），展开见典型问题/治理方案/代表案例；路口案例仅显示已固化方案的路口，结构为 认知→诊断→方案成效。
2. 提供认知经验（如「早高峰有空放」）→ 获取数据验证后：有数据→「已验证…」；无数据→「未找到数据支撑，已记录」；右下角 toast 含标签 + 已插入/已存在/已更新。
3. 重复提交同一认知 → toast 显示「已存在」；数据复核升级 → 「已更新」。
4. 提供诊断经验（如「附近四点放学/常有演唱会/早上特勤」）→「已吸收…并入库」；重复 → 「该诊断经验已存在（不再插入/更新）」。
5. 治理建议出现「参考依据」chip → hover 显示概要+ID；点击 → 跳转案例库对应场景/路口并高亮定位。
6. 原「经验吸收」可视卡片已消失，无重叠。
