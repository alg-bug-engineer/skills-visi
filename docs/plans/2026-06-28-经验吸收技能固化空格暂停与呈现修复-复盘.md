## 1. 背景

v2.0.5 已实现地图诊断阶段的**空格暂停**（`usePresentationPause` + `AnalysisQueue` 任务边界）。经验吸收（`skill_absorption`）与技能固化（`skill_build`）原先为 **SSE 同步直出**，未接入暂停。

## 3. 空格暂停 — 最终架构

### 3.1 设计原则

| 原则 | 说明 |
|------|------|
| 流式即时 | `thought_delta`、`file_delta`、`evidence` 等 **SSE 到达即 apply**，不进 AnalysisQueue |
| 边界 gate | 仅在 `stage_done`（吸收）、`stage_done` / `file_done`（固化）后入队 pause gate |
| 暂停缓冲 | `presentationPause.paused` 为 true 时事件写入 buffer，恢复后按序回放 |
| 回放逐帧 | buffer 中流式事件回放时使用 `requestAnimationFrame` 避免一帧刷完 |

### 3.2 模块入口

```
frontend-v2/src/utils/skillPresentationDispatch.ts   # 流式/边界分类 + pause gate 策略
frontend-v2/src/App.vue                            # dispatchSkillAbsorptionEvent / dispatchSkillBuildEvent
frontend-v2/src/composables/usePresentationBarrier.ts  # whenProcessAndVoiceSettled（固化用）
frontend-v2/src/utils/regressionSkillFlow.ts       # isSkillPresentationActive（空格键激活条件）
```

### 3.3 空格键激活条件

```text
docked && (
  loading || analysisQueue.isRunning || presentationPause.paused ||
  absorptionState.active ||
  (skillBuildState.visible && skillBuildState.status === 'running')
)
```

Toast 文案不变：**「分析暂停 · 空格继续」**（`WorkbenchLayout` · RT-UI-14）。

### 3.4 踩坑记录（必读）

#### Bug 1 — 吸收流程卡死（栅栏死锁）

- **现象**：面板显示「等待吸收追踪事件…」，固化抽屉空转  
- **原因**：`stage_start` 后调用 `whenPresentationSettled()` → 等待吸收面板无 `running` 行，但 `running` 行正是 `stage_start` 创建的；`stage_done` 在同队列后方无法执行  
- **修复**：`shouldEnqueueAbsorptionPauseGate` **仅** `stage_done`

#### Bug 2 — 先播语音再整段刷字（队列堆积）

- **现象**：吸收 / 写文件先长时间等待，然后内容一次性出现  
- **原因**：全部 SSE 进 AnalysisQueue，且在 `skill_absorption_start` / `file_created` 等处 `whenSettled` 阻塞；后端 delta 在队列中堆积  
- **修复**：流式事件同步 apply；仅阶段/文件结束处 enqueue pause gate

#### Bug 3 — 固化 gate 等待吸收 running 行

- **原因**：交错落盘时 `whenSettled` 含 `whenAbsorptionIdle`  
- **修复**：固化 pause gate 使用 `whenProcessAndVoiceSettled()`
