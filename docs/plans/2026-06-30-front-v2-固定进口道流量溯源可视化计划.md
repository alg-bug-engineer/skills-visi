# front-v2 固定进口道流量溯源可视化计划

## 目标

在 `frontend-v2` 中优化流量溯源地图表现：用户问题固定到某个进口道或转向（如北进口道、西左转道）后，前端只呈现该进口道对应的上游溯源树；沿路径向上游最多展示 5 跳，并用路径粗细表达流量占比，配合逐帧缩放、平移和全景收束。

## 参考实现要点

- 参考 `signalControl_agent-ui-design/static/corridor-coordination.html` 的干线诊断表达方式：路径按阶段揭示，主视图保留干线空间关系，关键路径用更强的线宽/高亮承载业务含义。
- 当前 `MapStage` 已有 `upstream_tree` storyboard、渐变蔓延、节点标注、初始拉远逻辑；本次不重写地图系统，只补强 storyboard 预处理、流量粗细和逐帧运镜。

## 改动计划

1. 新增 `src/utils/upstreamStoryboard.ts`
   - 解析固定进口道/转向提示。
   - 按目标进口道过滤 storyboard 的 `trees`。
   - 将节点/边限制在 `hop <= 5`。
   - 过滤帧中的 `reveal/focus`，避免超过 5 跳的节点和边进入可视化。
   - 提供 `upstreamEdgeStrokeWeight(flow_pct)`，把流量占比映射为稳定线宽。

2. 补充 TDD 单测 `src/utils/upstreamStoryboard.spec.ts`
   - 固定「西左转道」只保留西进口树。
   - 超过 5 跳的节点、边、帧不再出现。
   - 线宽随 `flow_pct` 增大且被上下限夹紧。

3. 优化 `src/components/MapStage.vue`
   - 在接收 `upstream_tree` 时先调用 storyboard 预处理。
   - 播放每一帧时应用 `center/zoom/fit`，让镜头跟随上游链路移动，最后自动 `setFitView` 收束全景。
   - 渐变路径按 `flow_pct` 加粗，最后一段显示方向箭头；多段线统一明暗，不只 dim 最后一段。
   - 标签中补充「流量 xx 辆/100」，和路径粗细语义一致。

4. 清理旧实现
   - 删除旧 `highlight_flow_sources` 一跳摘要相关死代码。
   - 删除已不再挂载的 `FlowTraceMapSummary` 与 `flowTraceCopy` 单元测试，避免旧验收约束干扰当前 storyboard 溯源。

## 验收

- `npm run test -- upstreamStoryboard upstreamFrame upstreamGradient upstreamLayout`
- `npm run test`
- `npm run build`
- 启动 `frontend-v2` dev server，确认页面可打开。
