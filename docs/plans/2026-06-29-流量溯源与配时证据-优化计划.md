# 流量溯源与配时证据 · 优化计划（2026-06-29）

## 背景与问题

用户验收反馈三类问题：

| # | 现象 | 根因 |
|---|------|------|
| 1 | 治理建议写「北左转绿灯仍有富余」，面板无对应指标 | `_spare_green_turn` 仅取 `green_utilization` 最低转向，**无阈值**、**headline 不写利用率数值**；前端主诊断卡未展示「过饱和 vs 富余」对比表 |
| 2 | 溯源结果难理解（多跳叠加%、按本路口转向拆分） | 与业务口径不符：应按**进口道**、**一跳上一路口**、**100 辆车来自上游左/直/右** 呈现 |
| 3 | 地图飞线 + 右侧大卡遮挡 | `highlight_flow_sources` 用中心点直线连接；`FlowTraceCard` 放在叙事右栏 |

## 目标口径（锁定）

### 配时主诊断（问题 1）

- 「绿灯仍有富余」**必须同时满足**：`green_utilization < low_utilization_diagnosis`（默认 0.60）。
- `primary_diagnosis.turn_balance` 显式携带过饱和方/富余方的 **饱和度 + 绿灯利用率**。
- Headline / evidence / 前端卡 **三处同数**，用户可核对。

### 流量溯源（问题 2）

**分析单元**：本路口 **进口道**（`f_dir8_no`），非「本路口某转向」。

**一跳定义**：该进口道上，取关联最强的 **上一路口** `cor_inter_id`（同进口各转向关联中的一跳锁定后，按 coverage 最大者）。

**100 辆车模型**（展示用）：

1. 假定该进口道过境车 **100 辆**；
2. 在上一路口，按 **左转 / 直行 / 右转**（`cor_turn_dir_no`）汇总一跳 coverage；
3. 在上一路口内部 **归一化为 100%** → `vehicles_of_100`（整数辆）；
4. 文案示例：「东进口约 100 辆过境车中，约 82 辆来自上一路口岔口，其中直行为主（82 辆）」。

> 原始 `flow_share_ratio` 仍保留在 `raw_coverage` 供研发核对；用户可见层只用「辆 / 100」。

### 地图与布局（问题 3）

- 溯源线 **沿渠化 link 折线**（`cognition.links[].path`），禁止中心飞线。
- **简要摘要卡**：地图 **左下角**（避让图例），每条进口道一行结论。
- 右侧叙事栈 **移除** 溯源大卡，避免与图例重叠。

## 验收标准

1. 演示路口晚高峰：主诊断若提「北左转富余」，卡片可见北左转 **绿灯利用率** 与东左转 **饱和度** 对比。
2. 溯源文案可读：按进口道、上一路口、左/直/右 **辆数** 表述，无「途经率可叠加」类术语面向用户。
3. 地图线为沿路折线；左下有简要摘要；右侧无溯源大卡。
