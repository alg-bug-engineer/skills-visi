# 领导汇报 · 叙事逻辑与地图呈现重构 — 需求理解

> 版本：2026-06-28 **v2（已确认方向）**  
> 状态：**已实现**（2026-06-28）  
> 关联：`PROJECT_OVERVIEW.md` · `地图语音暂停交互增强开发计划.md` · `2026-06-28-经验吸收与技能固化演示开发计划.md`

---

## 1. 背景

本系统已完成 **路口/干线拥堵诊断 → 治理建议 → 经验固化 → 下次复用** 的技术闭环。

**当前缺口**：领导看屏幕时，像进入一套「工程师后台」——步骤名偏技术、地图同一时刻叠了太多层、一线经验的价值埋在中段长文里，**故事线不清晰**。

---

## 2. 已确认约束（2026-06-28）

| # | 约束 | 含义 |
|---|------|------|
| **C1** | **汇报场景：单点 + 干线** | 两条完整用户路径都要能自然走通，不是只做单点 |
| **C2** | **界面不能像「演示」** | 禁止出现「报幕、第几幕、演示模式、专业模式、导演模式」等字样；领导应感觉这是**正在使用的业务系统**，不是排练过的 Demo |
| **C3** | **功能不可删** | 理解过程 8 步、语音播报、问题验证、规则诊断、治理建议、经验吸收、技能固化、配时环图、干线绿波、MetricStrip、InsightStack、空格暂停等**全部保留** |
| **C4** | **呈现可改** | 文案、UI 布局、地图动作时序、图标、播报顺序与摘要、信息主次与揭示节奏——均可调整 |

> **核心原则**：不是「做一套给领导看的假界面」，而是**把现有能力按业务故事重新编排**，让系统本身讲清楚价值。

---

## 3. 汇报对象与要感知的价值

### 3.1 主受众：非技术领导

| 领导关心的问题 | 系统应自然呈现的答案 |
|----------------|----------------------|
| 和以往有什么不同？ | 民警说口语，系统听懂并结构化 |
| 一线经验有没有浪费？ | 上次说的约束，下次自动带入 |
| 靠不靠谱？ | 有运行数据验证、有规则诊断 |
| 能不能推广？ | 经验写入技能包，同类场景复用 |
| 干线能不能管？ | 一条路问下来，发现堵点再深入单点 |

### 3.2 界面气质目标

- **像实战指挥台**，不像 PPT 翻页器
- **像智能体在干活**（理解过程打字、地图随推理变化、语音同步），不像有人在旁白报幕
- 所有能力都在，但**同一时刻只推一个结论**，其余随步骤自然出现

---

## 4. 业务故事线（对内编排，对外不显「幕」）

故事线在产品和开发文档里用于对齐节奏；**界面上不出现「第几幕」**，只保留现有「理解过程」步骤与地图联动。

```mermaid
flowchart LR
  A["民警描述问题"] --> B["锁定路口/道路"]
  B --> C["认识结构"]
  C --> D["数据验证"]
  D --> E["诊断与建议"]
  E --> F["经验写入复用"]
```

### 4.1 单点路径（8 步 → 故事映射）

| 理解过程步骤（保留） | 领导应感知到的业务含义 | 不可删的能力 |
|----------------------|------------------------|--------------|
| 理解问题 | 听懂民警原话与约束 | NLU、TTS `understand` |
| 匹配路口 | 找对路口；若有历史技能则自然提示 | 路口解析、Skill 命中 |
| 路口认知 | 看清结构、路名、关注/保护方向 | 渠化、轴路名、link 高亮 |
| 获取数据 | 拉取运行指标 | 数据抓取、MetricStrip（**延后**揭示） |
| 问题验证 | 用数据印证「确实堵」 | 证据卡、常发、InsightStack |
| 规则诊断 | 专家规则给出判断 | 规则引擎、FlowTimingGovernance |
| 生成建议 | 可执行治理方案 | 建议确认、SuggestionNote |
| 技能固化 | 经验吸收 + 写技能包 | absorption + skill_build |

### 4.2 干线路径（在单点之前）

| 阶段 | 领导应感知到的业务含义 | 不可删的能力 |
|------|------------------------|--------------|
| 理解道路与时段 | 「奥体西晚高峰哪里堵」被听懂 | corridor NLU、意图分类 |
| 沿路扫描 | 一条路上的问题路口一目了然 | corridor_scan 地图、侧栏列表 |
| 选定路口 | 用户点选，进入单点流水线 | corridor_pick → 单点 NLU |
| 后续 | 与单点 8 步相同 | 全流程 |

### 4.3 复用闭环（第二遍输入）

同一或同类路口再次提问时，**匹配路口**步骤内自然出现历史约束（如「垂直方向不能溢出」），无需单独「报幕式」横幅；语音与理解过程文案改写为业务口吻，**不展示**「演示」「样例」等字样。

## 5. 现状问题（基于代码与 UI 走查）

### 5.1 叙事与文案

| 问题 | 表现 |
|------|------|
| 步骤名偏技术 | 「获取数据」「规则诊断」「技能固化」 |
| 认知步信息过载 | 10 条 link 明细与地图同时出现 |
| 经验价值被淹没 | Skill 命中、历史约束夹在长段 technical 文本中 |
| 语音与面板认知分裂 | TTS 播摘要，面板仍刷全文 |

### 5.2 地图主舞台（信息同时堆叠）

同一时刻可能并存：POI 密集底图、多色 link、3D 渠化、HUD、证据 note、建议 note、MetricStrip、失衡横幅、图例、配时环/绿波小窗、InsightStack、工具栏按钮——**缺少「当前这一步只看什么」的秩序**。

截图基线：`artifacts/ui-screenshots/`（城市级 zoom + link 列表；渠化期多色叠加）

### 5.3 节奏

- 子 phase 切换过碎（saturation / direction / granularity…），地图 HUD 标题频繁跳变
- 空格暂停已有，但缺少与「理解过程当前步骤」对齐的**自然停顿点**

## 6. 需求范围

### 6.1 必须做

1. **步骤文案业务化**：改 `ANALYSIS_STEP_LABELS`、理解过程首行、`voice_narration.json` 引导语（8 步名称保留，表述贴近交警/指挥语境）
2. **地图呈现时序化**：按 pipeline phase / 理解步骤定义「主焦点层」，其余层**延迟揭示**而非删除
3. **单点 + 干线**两条汇报脚本走通，动作与文案连贯
4. **经验吸收/固化**：保留 L3 交错与抽屉，仅优化 trace 文案与右栏排版
5. **禁止报幕式 UI**：不出现进度条式的「故事幕」、不出现模式切换开关

### 6.2 明确不做

- 删除或关闭任何现有功能组件
- 新增「演示模式 / 专业模式」面向用户的切换
- 修改 orchestrator 状态机、Skill 匹配、规则引擎核心逻辑
- LLM 即兴生成播报文案（仍走 config + extractors）

### 6.3 可选（后续）

- Playwright 断言「步骤 N 时主舞台可见层 ≤ K」
- 常发日历热力图（问题验证阶段揭示）

## 7. 成功标准

| # | 标准 | 验证 |
|---|------|------|
| S1 | 领导 **3 分钟内**能复述：听懂 → 验证 → 建议 → 经验留存 | 试讲 |
| S2 | 界面 **无任何**「演示、报幕、第 N 幕、样例模式」字样 | UI 走查 |
| S3 | 功能清单 **100% 保留**（语音、8 步、固化、配时环、绿波、MetricStrip…） | 功能对照表 |
| S4 | 任意分析步骤，地图主焦点 **≤1 类结论 + 1 类结构**（如渠化 + 一条 HUD） | 截图 |
| S5 | 单点 + 干线脚本各走通一遍 | 彩排 |
| S6 | `bash scripts/regression.sh` 全绿 | 自动化 |

## 9. 已关闭的讨论项

| 原问题 | 确认结果 |

| Q1 演示入口 | **单点 + 干线** |
