# 单链路发光溯源重构实现计划

> **For Claude:** 参考 superpowers:executing-plans 逐任务实现；每个任务先写/改测试，跑通后提交。

**目标（Goal）：** 把当前"全路口多方向调试态"的流量溯源，重构为参考项目 `signalControl_agent-ui-design`（第二张图）那样的"单进口干线诊断"表达：用户给定进口/转向后只保留一条溯源链（≤5 跳），深色底图上用发光干线 + 沿线流动粒子 + 节点脉冲 + 极简标签呈现，去掉红色遮罩、虚线框、多方向同画与密集指标卡。

**状态：** 已实施（2026-06-30）；geom 拓扑一跳见 [流量溯源干线geom重构计划](./2026-06-30-流量溯源干线geom重构计划.md)。

**架构（Architecture）：**
- 后端收口：`_run_upstream_trace` 把"用户明示方向/转向"设为最高优先级，命中时只构建该方向树；否则才回退诊断命中的过饱和进口。storyboard 结构沿用现有（trees/edges/frames，已含 hop/saturation/flow_pct/approach）。
- 前端抽层：新建独立渲染模块 `UpstreamTraceLayer`（`src/lib/upstreamTraceLayer.ts`）封装 AMap 发光干线、流动粒子（rAF + setPosition）、节点脉冲、极简标签、dim、fit；`MapStage.vue` 只保留 storyboard 预处理 / 逐帧调度 / 运镜 / 渠化入场，删除旧的分段渐变线 + `.us-card` 密集卡 + 多方向 dim 实现。
- 纯函数（粒子插值、等距采样）放 `src/utils/traceParticles.ts`，可单测。

**技术栈：** 后端 Python (pytest)；前端 Vue3 + TS + AMap JS API 2.0 + Vitest + Playwright。配色对齐参考项目（琥珀干线 `#ffcf7a` / glow `rgba(245,166,35,.22)`、粒子 `#fff4d6`、目标红 `#ff4757`、上游琥珀 `#f5a623`、治理绿 `#2ed573`、标签底 `rgba(6,12,24,.94)`）。

## Task 4：渲染模块 UpstreamTraceLayer

**Files:**
- Create: `frontend-v2/src/lib/upstreamTraceLayer.ts`

**API：**
```
class UpstreamTraceLayer {
  constructor(AMap, map)
  reset()
  hasOverlay(id): boolean
  revealEdge(id, path, { dim, weight })   // glow 外层 + core 亮线 + 2~3 流动粒子
  revealNode(id, lon, lat, { role, dim, color })  // .us-node 脉冲 marker
  revealLabel(id, lon, lat, html, offset)  // .us-label 极简标签 marker
  overlays(): AMapOverlay[]   // 供 setFitView
  dispose()
}
```
粒子用内部单一 rAF 循环 + `setPosition`（参考项目实现，不用 moveAlong）；`reset/dispose` 取消 rAF。所有 overlay 进 `byId` 注册表，幂等。
