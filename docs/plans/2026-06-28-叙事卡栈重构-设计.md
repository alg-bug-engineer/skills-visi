## 1. 目标（来自用户 4 点诉求）

1. **统一生长卡栈**：定位路口后，右上角出现一张随叙事向下生长的卡片。顶部是路口认知（身份+结构），理解过程推进时把 **饱和度→四向指标→失衡→干线绿波→投诉/常发** 逐项 ✓ 追加到下方；删除右下角运行数据浮层，数据全部迁入这张卡。再往下自然接 **问题验证（出现后自动折叠）→ 治理建议**。整列从上到下就是完整故事线。
2. **地图动画连贯**：镜头只前进不回跳（市域→路口→车道级，zoom 单调递增/保持）；步骤切换只做平移/微调，不重新从最原始下钻；用户手动拖拽/缩放后保留视角，不被下一步强制拉回。
3. **问题验证/治理建议** 调到右上角卡栈内，问题验证出现后自动折叠，治理建议在折叠条目下方出现（层级逻辑）；两者样式参考 `artifacts/bugs/列表样式.png`（深色 + 青蓝勾选行，与"智能体推理"一致）。
4. **复盘**：1-3 完成后产出面向领导的叙事/播报/地图一致性复盘文档。

## 2. 约束（沿用 v2）

- 功能不可删（C3）：理解过程 8 步、语音、固化、配时环、绿波、问题验证、治理建议全部保留。
- 无演示壳（C2）：不出现"演示/报幕/第 N 幕"。
- 右侧"理解过程/智能体推理"栏 **保留并存**；新卡栈浮在地图上。

## 4. 新组件 `IntersectionNarrativeStack.vue`

绝对定位：地图区右上角（`stage-body` 内 `position:absolute; top/right`），在右侧理解过程栏之左。**只增不减**，避免阶段切换闪烁（沿用 ChannelizationLegend 的 reveal 粘性模式）。

段落与门控（`usePresentationSequence.focusStepIndex` + `phase`）：

| # | 段落 | 出现时机 | 数据源 |
|---|------|----------|--------|
| 1 | 认知头：路口名 / inter_id / 臂·道 / zoom 实时 / 当前层级 / 关注·保护方向 / 下钻提示 | 定位（COGNITION 步） | `cognition`、地图 LOD/zoom |
| 2a | 运行数据·饱和度 | DATA_FETCH `saturation` | `runtimeMetrics`/`metrics_by_arm` |
| 2b | 运行数据·四向指标 | DATA_FETCH `direction`/`granularity` | `metrics_by_arm` |
| 2c | 运行数据·失衡 | DATA_FETCH `imbalance` | `evidence`/metrics |
| 2d | 运行数据·干线绿波 | DATA_FETCH `corridor` / RULE | `evidence.corridor_context` |
| 2e | 运行数据·投诉/常发 | PROBLEM_EVIDENCE | `evidence`（chronic/complaint） |
| 3 | 问题验证（出现后自动折叠为 ▾ 摘要行） | PROBLEM_EVIDENCE | `evidence`（reasons） |
| 4 | 治理建议（折叠行下方） | SUGGESTION | `governanceSuggestion` |

- 实时 zoom/当前层级：`MapStage` 暴露当前 zoom 与 LOD 档（路网/轮廓/车道），通过事件/prop 传入卡。
- 列表样式：段落 2/3/4 用统一的 `✓ 行` 列表（青蓝勾、深底）。

## 5. 地图连贯（`MapStage.vue`）

- 合并 `drillToIntersection`（14→16.2→17.5）与 `enterChannelizationView`（18.5）为**单调递增**的连续 fly；进入渠化态不再 `clearOverlays` + 二次下钻。
- `channelizationLocked` 切换：从 link 态平滑过渡到渠化态，不闪。
- `applyMapScene`/`update_metrics`：渠化态下不再每步 `setCenter` 回拉中心；只更新覆盖物。
- 新增 `userInteracted` 标志：监听 `dragend`/`zoomend`（用户手势）后置位，后续步骤跳过强制 recenter / fly，只同步覆盖物。新一轮分析 `prepareNewAnalysisRun` 时复位。
