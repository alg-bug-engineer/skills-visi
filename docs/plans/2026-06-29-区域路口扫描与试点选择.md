# 区域路口扫描与试点选择 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 新建 `region-scan/` 项目，对全区域信号路口逐路口跑完整诊断（信控视角），产出快照，并用高德地图呈现全域体检结果与试点推荐榜，支撑领导审批选点。

**Architecture:** 复用 `backend/intersection_agent` 的 PG 访问、认知、诊断、信控治理服务；新增"区域批量扫描引擎 → 快照存储 → 扫描 API → 高德前端"四层。批处理离线跑、写快照；地图只读快照。

**Tech Stack:** Python 3.13（async, asyncpg via 现有 `PostgresPool`）、pytest、FastAPI、Vite + Vue 3 + 高德 JS API。

**配套设计文档：** `docs/plans/2026-06-29-区域路口扫描与试点选择-design.md`

## 前置说明（执行者必读）

- 本项目**依赖现有 backend 包** `intersection_agent`，须在能 import 它的同一 Python 环境中运行（共用 `backend` 的 venv / `uv`）。
- 涉及调用现有服务时，**先 Read 对应服务文件确认方法签名再写代码**，不要凭猜。关键服务：
  - `backend/intersection_agent/services/intersection_cognition_service.py` (`IntersectionCognitionService.fetch`)
  - `backend/intersection_agent/services/problem_evidence_service.py` (`ProblemEvidenceService.build`)
  - `backend/intersection_agent/services/rule_engine.py` (`RuleEngine.diagnose_comprehensive`)
  - `backend/intersection_agent/services/flow_timing_governance_service.py` (`FlowTimingGovernanceService.build`)
  - `backend/intersection_agent/services/corridor_scan_service.py`（区域枚举与批量指标的范本）
  - `backend/intersection_agent/utils/corridor_metrics.py`、`utils/data_window.py`
- DB 连接走现有 `intersection_agent.config.get_settings()` + `.env`（PGHOST 等）。**绝不**在新代码硬编码连接串。
- 每个 Python 任务遵循 TDD：先写失败测试 → 跑失败 → 最小实现 → 跑通过 → commit。`MOCK_DB=1` 用于不连库的单测。

## Phase 3 · 分层与试点评分（核心分析价值）

### Task 3.1: problem_band 分层

**Files:**
- Create: `region-scan/region_scan/classify.py`
- Test: `region-scan/tests/test_classify.py`

**口径（集中在一处 + 测试）：**
- `工程可解`：饱和度持续 ≥ 过饱和阈值（`thresholds.yaml` 的 `saturation.oversaturation`）且信控改善上限低。
- `配时可解`：检出失衡 / 空放 / 绿灯利用率低 / 相位问题，且未达过饱和、信控改善上限高。
- `无明显问题`：无显著问题且饱和度适中。
- `数据不足`：`has_data=False`。

**Step 1: 失败测试** — 四个 case 各一个断言（过饱和→工程可解；高失衡未饱和→配时可解；平稳→无明显问题；无数据→数据不足）。

**Step 2:** 跑失败。

**Step 3: 实现** `def classify_problem_band(diag: dict, thresholds: dict) -> str`。

**Step 4:** 跑通过。

**Step 5: Commit** `feat(region-scan): problem_band 分层口径`

## 验收标准（对应设计文档 §10）
1. 一条命令完成全区域扫描并生成快照。
2. 高德地图展示全部有坐标的信号路口，按时段/指标/分层着色正确。
3. 点击任一路口可见完整认知 + 信控诊断 + 治理建议。
4. 试点推荐榜按潜力排序并与地图联动。
5. 过饱和路口被正确标注"工程可解 / 配时无效"。
6. 关键分层与评分逻辑有单元测试。
