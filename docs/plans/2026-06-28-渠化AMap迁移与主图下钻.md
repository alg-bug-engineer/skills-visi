# 渠化 AMap 迁移与主图下钻 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 把项目内渠化渲染从 THREE.js(3D) 与 D3/SVG(2D) 全部迁移到「基于高德地图(AMap)矢量覆盖物」的实现，并整合进主地图随缩放下钻，所有标注/图标/文本框/强调/虚线零出错。

**Architecture:** 新建一个纯函数渠化几何库 + 一个 AMap 覆盖物渲染器（`ChannelizationAmapLayer`），把现有 `channelizationLayer.js` 的全部能力（静态渠化 + 4 套阶段标注）逐一映射为 AMap 覆盖物；在 `MapStage` 主地图上按 zoom 分级下钻，移除独立 3D 舞台；迷你窗改用小型 AMap 实例。阶段→标注映射（原 `ChannelizationCanvas3D.applyPhaseHighlight`）原样保留，确保与 voice/presentation 时序一致。

**Tech Stack:** Vue 3 + TS、AMap JS API v2.0（`utils/amap.ts` 已封装 loader）、Vitest 单测、现有 `cognitionChannelAdapter.ts` 数据适配层。

**阶段→标注映射（必须 1:1 保留，见 `ChannelizationCanvas3D.applyPhaseHighlight`）：**
- 结构阶段（`COGNITION_STRUCTURE_PHASES`：links/channelization）：只画静态渠化，不叠标注，不画排队车。
- `highlightTurn` 存在且非结构阶段 → `applyTurnHighlight`（车道色带 + 黄环 + 文本框）。
- `saturation` / `granularity` → `applyCheckHighlight('saturation', verdict, ev)`（路面强调面 + 排队色带 + 末端线 + 文本框）。
- `imbalance` → `applyCheckHighlight('imbalance', …)`（进出口失衡色带）。
- `traffic` → 不叠饱和度浮标（仅排队车）。
- `direction` → `applyDirectionRoleHighlight(focus, protect)`（关注橙红/保护绿/其他 dim）。
- 多阶段 → `applyArmSceneLabels`（臂外缘文本框）。

**必须逐一复刻的覆盖物清单（THREE → AMap 映射表）：**

| 元素 | THREE 现状 | AMap 目标 |
|---|---|---|
| 路面 | PlaneGeometry | `AMap.Polygon`（填充） |
| 缘石/中心黄实线 | Line | `AMap.Polyline`（实线，黄色双线） |
| 车道分隔虚线 | zDash LineSegments | `AMap.Polyline` `strokeStyle:'dashed'` `strokeDasharray` |
| 停止线 | Plane+Line（红） | `AMap.Polyline`（粗白/红） |
| 斑马线 | Plane 组 | `AMap.Polygon` 条带组 |
| 转向箭头 | makeArrow 三角网格(B/C/D/A 组合) | `AMap.Marker` + SVG（`arrowSvg` 已支持组合，需补 A 掉头与多分支共杆） |
| 转角圆弧 | 贝塞尔 Line | `AMap.Polyline`（二次贝塞尔采样点） |
| 排队车 | Box 模型 | `AMap.Marker`(SVG 车) 或 `AMap.Polygon` 小块（按 LOD） |
| 强调面（check/turn/direction） | 加色 Plane（AdditiveBlending） | `AMap.Polygon`（半透明 fill；无叠加混合，用配色补偿） |
| 排队/信控/失衡色带 | Plane | `AMap.Polygon` |
| 黄色高亮环（turn） | RingGeometry | `AMap.CircleMarker` 或 `AMap.Marker`+SVG 环 |
| 文本框（标注/指标/verdict/臂标签） | CanvasTexture Sprite（圆角底 + 双行文字） | **自定义 HTML `AMap.Marker`**（`content` = 圆角 div，双行 + 颜色边框，复刻 `_makeTextTex` 样式） |

## 关键约定

- **坐标**：所有局部米→经纬度统一走 `metersToLngLat(center, u, v, bearing)`（demo 已验证，含右行修正：进口 -v 左、出口 +v 右）。
- **臂方位角**：优先用真实 `geom`（`armAngleFromLink`），缺失时回退 `entrance_angle` / dir 映射。
- **zIndex 顺序**：路面(14) < 出口面(15) < 强调面/色带(16) < 斑马线(17) < 虚线/转角(18) < 黄线(19) < 停止线(20) < 文本框/箭头(30+)。
- **下钻 LOD**：`zoom<16` 路网；`16≤zoom<18` 轮廓（真实中心线 + 路面面）；`zoom≥18` 全渠化 + 阶段标注。阈值集中在一处常量。
- **零出错铁律**：每个覆盖物创建后必须 `map.add`；每次 rebuild/phase 切换前必须清理上一批（保留分组数组，先 `remove` 再重建），杜绝泄漏与叠加重影。所有 `AMap.*` 调用前确认 `AMap` 已 load（复用 `loadAmap()`）。

- **回滚**：每 Phase 独立 commit；保留旧文件直到 Phase 5 才删，前 4 phase 可随时切回。
