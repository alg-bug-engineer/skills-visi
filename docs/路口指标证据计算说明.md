# 路口指标证据计算说明

> **文档定位**：说明智能体「问题验证证据」（`problem_evidence`）及相关约束量化所涉及的数据表、SQL、指标口径与判定逻辑。  
> **代码真源**：`backend/intersection_agent/services/problem_evidence_service.py`  
> **阈值真源**：`backend/rules/thresholds.yaml`  
> **关联服务**：`data_fetcher.py`（规则诊断查数）、`intersection_cognition_service.py`（分向饱和度补充）、`constraint_resolver_service.py`（约束量化）

---

## 1. 计算范围与输出结构

### 1.1 触发时机

诊断流水线在 `data_fetch` + `intersection_cognition` 完成后，由 `ProblemEvidenceService.build()` 生成 `meta.problem_evidence`，用于：

- 验证用户描述的拥堵是否**常发性**
- 验证是否**周期性**（如每周三晚高峰）
- 输出**量化运行指标**（排队、停车、饱和度、溢流风险等）
- 输出**分方向画像**（`by_direction`），供约束量化使用

### 1.2 输出 JSON 字段一览

| 输出块 | 字段 | 含义 |
|--------|------|------|
| `chronic` | `is_chronic`, `congested_days`, `congested_dates`, `verdict` | 常发性拥堵判定 |
| `dow_pattern` | `hit_days`, `hit_rate`, `target_dow`, `verdict` | 星期规律判定 |
| `metrics` | `avg_delay_s`, `avg_queue_m`, `max_queue_m`, `saturation_rate`, `imbalance_index`, `spillback_risk_max` 等 | 路口级汇总指标 |
| `by_direction` | `group`, `avg_queue_m`, `saturation`, `queue_storage_ratio`, `focused` | 分方向（东西向/南北向等） |
| `thresholds_used` | 各判定阈值快照 | 便于前端/审计复现 |
| `source_tier` | `dwd_rolling_7d` / `dws_weekday_pattern` / `none` | 数据覆盖层级 |
| `query_trace` | 实际执行的 SQL 与行数 | 调试与联调 |

## 2. 时间窗口与查询参数

所有查询共用 `build_data_window()` 解析结果（`utils/data_window.py`）。

### 2.1 日历窗（DWD 表）

| 参数 | 说明 | 示例（晚高峰，reference_date=2026-06-14） |
|------|------|------------------------------------------|
| `date_from` ~ `date_to` | 近 7 日滚动窗（含 reference_date） | 2026-06-08 ~ 2026-06-14 |
| `time_slot` | NLU 时段，`stat_time::time` 过滤 | 16:00–18:00 |
| `dow_filter` | 工作日高峰默认 Mon–Fri | `[1,2,3,4,5]` |

### 2.2 周模式窗（DWS 表）

| 参数 | 说明 | 示例 |
|------|------|------|
| `step_index` | 5 分钟时间片，`step_index * 5 = 距 0:00 分钟数` | 晚高峰 16:00–18:00 → **192–215** |
| `day_of_week` | 1=周一 … 7=周日 | 有 DWD 时用 `dow_filter`；无 DWD 时降级为 `primary_dow`（提问日星期） |

### 2.3 Schema 与环境变量

| Schema | 环境变量 | 用途 |
|--------|----------|------|
| `xianchang` | `PG_FLOW_SCHEMA` | DWD/DWS 运行指标 |
| `road6` | `PGSCHEMA` | 路口维表、渠化（认知阶段，非证据主表） |

## 4. 核心 SQL 与指标计算

以下 SQL 与代码一致；参数占位符 `$1`… 对应该服务传入的 `inter_id`、`date_from`、`date_to`、`slot_start`、`slot_end`、`dow_filter`、`step_start`、`step_end`。

### 4.1 按日 DWD 聚合 — 常发性 / 星期规律

**表**：`xianchang.dwd_tfc_inter_dir_perf_5min`  
**label**：`evidence_daily_dwd`

```sql
SELECT stat_time::date AS stat_date,
       EXTRACT(ISODOW FROM stat_time)::int AS dow,
       AVG(stop_time) AS avg_stop_time,
       AVG(queue_len_avg) AS avg_queue_m,
       MAX(queue_len_max) AS max_queue_m,
       AVG(delay_index) AS delay_index
FROM xianchang.dwd_tfc_inter_dir_perf_5min
WHERE inter_id = $1
  AND is_deleted = 0
  AND stat_time::date BETWEEN $2 AND $3          -- 近7日
  AND stat_time::time >= $4 AND stat_time::time < $5  -- 时段，如 16:00–18:00
GROUP BY stat_date, dow
ORDER BY stat_date;
```

**使用到的源字段**：

| 源字段 | 含义 |
|--------|------|
| `stop_time` | 平均停车时间（秒） |
| `queue_len_avg` | 平均排队长度（米） |
| `queue_len_max` | 最大排队长度（米） |
| `delay_index` | 延误指数（DWD 量纲，证据层仅展示，不参与规则阈值 1.5 对齐） |

**派生指标与判定**：

| 派生指标 | 计算方式 | 用途 |
|----------|----------|------|
| `chronic.congested_days` | 满足「超标日」条件的天数 | 常发性 |
| `chronic.is_chronic` | `congested_days >= 4`（默认） | 常发性结论 |
| `chronic.congested_dates` | 超标日期列表 | 前端日历/审计 |
| `dow_pattern.hit_days` | 目标星期几中超标天数 | 周期性 |
| `dow_pattern.hit_rate` | `hit_days / total_days` | 周期性 |
| `metrics.avg_delay_s` | 所有 daily_rows 的 `avg_stop_time` 均值 | 路口级展示 |

**单日「超标」判定**（`_is_congested_day`，满足任一即为超标）：

| 条件 | 阈值（thresholds.yaml） |
|------|-------------------------|
| `avg_stop_time >= excess_delay_s` | 60 s |
| `max_queue_m >= long_queue_m` | 100 m |
| `saturation_max >= saturation.high` | 0.80（来自 DWS evaluation 行，全日共用） |
| `avg_queue_m >= long_queue_m * 0.8` | 80 m |
| `queue_storage_ratio >= queue_storage_ratio_high` | 0.80（DWD 日聚合行若含该字段） |

### 4.2 分方向 DWS 聚合 — 排队 / 溢流代理

**表**：`xianchang.dws_inter_dir_turn_perf_5min_mm`  
**label**：`evidence_direction_dws`

```sql
SELECT f_dir_8_label,
       AVG(queue_len_avg) AS avg_queue_m,
       MAX(queue_len_max) AS max_queue_m,
       AVG(stop_time) AS avg_stop_time,
       AVG(delay_index) AS delay_index,
       AVG(queue_len_avg / NULLIF(rid_length_m, 0)) AS queue_storage_ratio
FROM xianchang.dws_inter_dir_turn_perf_5min_mm
WHERE inter_id = $1
  AND day_of_week = ANY($2::int[])    -- 有DWD: [1..5]; 无DWD: [primary_dow]
  AND step_index BETWEEN $3 AND $4    -- 如 192 AND 215
  AND is_deleted = 0
GROUP BY f_dir_8_label;
```

**使用到的源字段**：

| 源字段 | 含义 |
|--------|------|
| `f_dir_8_label` | 八方向标签（东/西/南/北 等） |
| `queue_len_avg` / `queue_len_max` | 排队长度 |
| `stop_time` | 停车时间 |
| `rid_length_m` | 进口道/RID 长度（米），用于储车比 |

**派生指标**：

| 输出字段 | 公式 / 聚合 | 说明 |
|----------|-------------|------|
| `queue_storage_ratio` | `AVG(queue_len_avg / rid_length_m)` | 排队占进口道长度比例，**溢流风险代理** |
| `spillback_risk` | **等于** `queue_storage_ratio` | 代码中直接 proxy，无二次换算 |
| `metrics.queue_storage_ratio_max` | 各方向 `queue_storage_ratio` 的 **MAX** | 路口级最差方向 |
| `metrics.spillback_risk_max` | 同上 | 与 `queue_storage_ratio_max` 相同 |
| `metrics.avg_queue_m` | DWD 缺失时，各方向 `avg_queue_m` 均值 | 降级 |
| `metrics.max_queue_m` | DWD 缺失时，各方向 `max_queue_m` 最大值 | 降级 |
| `by_direction[].*` | 按 `F_DIR8_TO_GROUP` 映射到东西向/南北向等后 **再聚合** | 见 §5 |

> **说明**：代码未过滤 `turn_dir_no = 0`（方向聚合行）。若需仅统计方向级、不含分转向重复，可手工加 `AND turn_dir_no = 0`（联调 SQL 常用此条件）。

### 4.3 路口评价 DWS — 饱和度 / 失衡

**表**：`xianchang.dws_inter_evaluation_5min_mm`  
**label**：`evidence_saturation_dws`

```sql
SELECT AVG(saturation_max) AS saturation_max,
       AVG(saturation_avg) AS saturation_avg,
       AVG(unbalance_index) AS imbalance_index
FROM xianchang.dws_inter_evaluation_5min_mm
WHERE inter_id = $1
  AND day_of_week = ANY($2::int[])
  AND step_index BETWEEN $3 AND $4
  AND is_deleted = 0;
```

**派生指标**：

| 输出字段 | 来源 | 说明 |
|----------|------|------|
| `metrics.saturation_rate` | `saturation_max`（或 DataFetcher 的 turn_saturation） | 路口饱和度 |
| `metrics.imbalance_index` | `unbalance_index` | 进口服务失衡 |
| 常发性降级 | `saturation_max >= 0.80` 且无 DWD | `method: dws_pattern_estimate` |

## 5. 分方向聚合规则

DWS 返回的 `f_dir_8_label`（如「东」「西」）映射为方向组：

| f_dir_8_label | 方向组 |
|---------------|--------|
| 东 / 西 / 东向西 / 西向东 | 东西向 |
| 南 / 北 / 南向北 / 北向南 | 南北向 |
| 其他八方向标签 | 对应斜向组 |

每组内对多条进口道记录做：

| 字段 | 组内聚合 |
|------|----------|
| `avg_queue_m` | MEAN |
| `max_queue_m` | MAX |
| `avg_delay_s` | MEAN(`avg_stop_time`) |
| `queue_storage_ratio` | MAX |
| `saturation` | MAX（来自 cognition 的 `direction_groups`） |
| `focused` | 是否落在 NLU `directions` 解析出的关注组 |

## 6. 数据覆盖降级策略

```
有 DWD 日历数据？
  ├─ 是 → source_tier = dwd_rolling_7d
  │       常发性/星期规律：真实日历统计
  │       DWS 查询 day_of_week = dow_filter (工作日 1-5)
  │
  └─ 否 → 有分向 DWS？
            ├─ 是 → source_tier = dws_weekday_pattern
            │       常发性：仅当 saturation_max >= 0.8 时估算为常发
            │       星期规律：无法算 hit_rate，文案降级
            │       DWS 查询 day_of_week = primary_dow（提问日）
            │
            └─ 否 → source_tier = none，证据为空/不足
```

**coverage_warning 文案**：`近7日 DWD 明细无数据，常发性/星期规律基于 DWS 周模式估算`

## 7. 约束量化（溢流风险与 0.05）

约束量化**不再查库**，使用 §4.2 算出的 `by_direction[].queue_storage_ratio`。

**配置**（`thresholds.yaml`）：

```yaml
spillback:
  risk_high: 0.80          # 绝对红线
queue:
  queue_storage_ratio_high: 0.80
constraint:
  spillback_margin: 0.05   # 允许恶化 5 个百分点
  queue_growth_margin: 0.10
  saturation_margin: 0.10
chronic:
  min_congested_days: 4
  window_days: 7
  dow_hit_rate_high: 0.75
delay:
  excess_delay_s: 60
queue:
  long_queue_m: 100
saturation:
  high: 0.80
```

**用户说「垂直方向不能溢出」时**：

```
保护方向 = 与主方向正交的 direction group（如主向东西向 → 保护南北向）
baseline   = 保护方向当前的 queue_storage_ratio
cap        = min(baseline + spillback_margin, spillback.risk_high)
           = min(baseline + 0.05, 0.80)
```

输出示例：`南北向溢流风险不超过 0.15（当前约 0.10，上限 0.80）`

## 8. 同流水线 DataFetcher 查表（补充 metrics）

`ProblemEvidenceService._aggregate_metrics()` 会合并 `data_payload` 中 DataFetcher 的结果：

| 表 | SQL 要点 | 写入 evidence |
|----|----------|---------------|
| `dwd_tfc_inter_dir_perf_5min` | `AVG(delay_index)`，近7日+时段+`dow_filter` | 规则用 `delay_index`；证据展示可来自 evaluation |
| `dws_inter_evaluation_5min_mm` | `AVG(saturation_max)`, `AVG(unbalance_index)` | `metrics.saturation_rate`, `metrics.imbalance_index` |
| `dws_turn_saturation_5min_mm` | `MAX(turn_saturation)` | `traffic_flow.saturation_rate` → 证据饱和度 |
| `dws_turn_green_utilization_5min_mm` | `AVG(green_utilization)` | 规则诊断，不直接进 evidence |
| `dwd_ctl_inter_plan_cfg` + `dwd_ctl_inter_plan_stage_timing` | 周期、绿信比 | 规则诊断 |
| `road6.dwd_tfc_rltn_wide_inter_ft_link` | 渠化 | 规则诊断 |

DataFetcher DWD 延误 SQL：

```sql
SELECT AVG(delay_index) AS delay_index,
       COUNT(*)::int AS sample_count
FROM xianchang.dwd_tfc_inter_dir_perf_5min
WHERE inter_id = $1 AND is_deleted = 0
  AND stat_time::date BETWEEN $2 AND $3
  AND stat_time::time >= $4 AND stat_time::time < $5
  AND EXTRACT(ISODOW FROM stat_time)::int = ANY($6::int[]);
```

## 9. 路口认知分向饱和度（补充 by_direction.saturation）

**表**：`road6.dwd_tfc_rltn_wide_inter_ft_link` JOIN `xianchang.dws_turn_saturation_5min_mm`

```sql
SELECT l.link_id, l.dir4_label, l.dir8_label,
       AVG(s.turn_saturation) AS saturation,
       MAX(s.turn_saturation) AS saturation_max
FROM road6.dwd_tfc_rltn_wide_inter_ft_link l
LEFT JOIN xianchang.dws_turn_saturation_5min_mm s
  ON l.inter_id = s.inter_id AND l.link_id = s.link_id
 AND s.day_of_week = ANY($2::int[])
 AND s.step_index BETWEEN $3 AND $4
 AND s.is_deleted = 0
WHERE l.version_id = $5 AND l.inter_id = $1
  AND (l.link_role = 'entrance' OR l.link_role = '进口')
GROUP BY l.link_id, l.dir4_label, l.dir8_label;
```

结果经 `direction_groups` 汇总后写入 `by_direction[].saturation`。

## 10. 联调示例 SQL

以 **奥体中路与新泺大街路口**（`011wwe295cm00001`）晚高峰为例：

```sql
SET search_path TO road6, xianchang, public;

-- 路口信息
SELECT inter_id, inter_name FROM road6.dim_inter_info
WHERE inter_id = '011wwe295cm00001' AND version_id = '20260501';

-- 证据：按日 DWD（2026-06-08 ~ 2026-06-14, 16:00-18:00）
SELECT stat_time::date AS d,
       AVG(stop_time) AS avg_stop,
       MAX(queue_len_max) AS max_q
FROM xianchang.dwd_tfc_inter_dir_perf_5min
WHERE inter_id = '011wwe295cm00001' AND is_deleted = 0
  AND stat_time::date BETWEEN '2026-06-08' AND '2026-06-14'
  AND stat_time::time >= '16:00' AND stat_time::time < '18:00'
GROUP BY 1 ORDER BY 1;

-- 证据：分方向 DWS（工作日, step 192-215）
SELECT f_dir_8_label,
       ROUND(AVG(queue_len_avg)::numeric, 1) AS avg_q,
       ROUND(MAX(queue_len_max)::numeric, 1) AS max_q,
       ROUND(AVG(queue_len_avg / NULLIF(rid_length_m, 0))::numeric, 3) AS queue_storage_ratio
FROM xianchang.dws_inter_dir_turn_perf_5min_mm
WHERE inter_id = '011wwe295cm00001' AND is_deleted = 0
  AND day_of_week = ANY(ARRAY[1,2,3,4,5])
  AND step_index BETWEEN 192 AND 215
GROUP BY f_dir_8_label;

-- 证据：路口评价
SELECT ROUND(AVG(saturation_max)::numeric, 3) AS sat,
       ROUND(AVG(unbalance_index)::numeric, 3) AS imbalance
FROM xianchang.dws_inter_evaluation_5min_mm
WHERE inter_id = '011wwe295cm00001' AND is_deleted = 0
  AND day_of_week = ANY(ARRAY[1,2,3,4,5])
  AND step_index BETWEEN 192 AND 215;
```

## 11. 指标输出对照表（速查）

| 前端/证据字段 | 主要来源表 | 计算口径 |
|---------------|------------|----------|
| `chronic.is_chronic` | `dwd_tfc_inter_dir_perf_5min` | 近7日超标日 ≥ 4 |
| `dow_pattern.hit_rate` | 同上 | 目标星期超标比例 |
| `metrics.avg_delay_s` | DWD `stop_time` 日均再均值 | 秒 |

| `metrics.imbalance_index` | `dws_inter_evaluation` | 失衡系数 |

| `by_direction[].queue_storage_ratio` | 同上，按方向组 MAX | 溢流代理 |
| `quantitative_constraints` | 内存计算 | baseline + 0.05，cap ≤ 0.80 |
