# 路口控制策略参考规则

## 文档依据
- 路口主策略包：失衡纠偏、特殊时段、结构性防溢流、高冲突安全
- 专项叠加：公交、慢行、停车诱导、事件应急、数据降级

## 通用护栏
- 所有结论必须引用供给、需求、状态或执行结果中的证据；缺少证据时输出需要补采的数据。
- 优先级顺序：安全底线、防溢流防锁死、基本服务公平、效率、体验。

## 路口级策略包

路口策略应服务于“绿信比、相位相序、防溢流、慢行保护”四类任务。控制策略阶段只输出策略指令，不直接生成配时秒数。

### 失衡纠偏策略包

- 适用场景：普通路口平峰、夜间或方案老化，表现为空放、相位利用率差异大、绿信比分配失衡、周期过长或过短、相位相序与渠化不匹配。
- 主策略动作：短周期运行、削减空放、重构绿信比、请求式触发、按转向流量微调相序。
- 约束要求：采用小步长调整，保留支路最小服务，避免因追求效率导致某一进口长期受抑制。

### 特殊时段保障策略包

- 适用场景：学校上下学、医院早间就诊、商圈晚高峰、景区或枢纽短时集散。
- 主策略动作：专用时段方案、行人专用或二次过街、右转控转、关键方向短时增绿、公交到站优先、特殊时段结束后快速回切。
- 约束要求：慢行安全与接送秩序优先，不能长期保留特殊高峰策略造成平峰空放。

### 结构性瓶颈防溢流策略包

- 适用场景：短车道、近间距串联、进口出口不匹配、桥隧上下游、下游阻塞或连续周期不清空。
- 主策略动作：出口优先、量出为入、溢流方向刹车、相位重构、关键转向清空、同步输出车道功能或物理改造建议。
- 约束要求：一旦出现下游余量不足或排队接近存储上限，应从效率优先切换为防锁死优先。

### 高冲突安全保护策略包

- 适用场景：老城区、学校周边、商业街区、慢行活跃路口、恶劣天气和右转机非人冲突高发路口。
- 主策略动作：行人和非机动车分离放行、黄灯和全红清空延长、右转限行或分时放行、行人绿延长、安全岛等待保护。
- 约束要求：安全优先于机动车效率，涉及冲突风险时应要求人工复核或现场确认。

## 专项叠加规则

- 公交优先专项：适用于公交骨干走廊、枢纽集散、大客流活动散场，动作包括公交到站优先、专用相位、站点前后路口联动放行。
- 行人和非机动车保护专项：适用于学校、医院、商圈、景区和慢行高需求街区，动作包括行人专用相位、二次过街、非机动车提前放行和右转冲突抑制。
- 停车与诱导联动专项：适用于医院、商圈、景区、枢纽、老城区停车矛盾突出区域，动作包括停车余位诱导、外围停车换乘、出入口分时控制和违停敏感路段保护。
- 事件应急专项：适用于事故、施工、恶劣天气、大型活动和临时管制，动作包括替代路径优先、远端截流、近端疏散、安全参数上调和交警导航联动。
- 数据异常与能力降级专项：适用于检测器离线、通信异常、控制机能力不足或执行状态不可信，动作包括回退最近验证方案、冻结高频调节项、保留最小安全和基本服务控制。

## 策略切换规则

- 策略选择按“先定主层级、再判问题性质、最后叠加专项策略”的顺序执行。
- 可用多时段固定方案解决的问题，不应过度升级为复杂自适应；可用柔性协调解决的问题，不应直接强截流。
- 出现溢流、锁死、下游阻塞、高冲突或异常事件时，目标自动切换为安全和防扩散优先。
- 进入和退出应采用不同阈值，并设置最小驻留时间和渐进恢复机制，避免在阈值附近频繁抖动。
