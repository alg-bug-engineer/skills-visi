# 路口方案生成参考规则

## 文档依据
- 配时方案字段语义
- 相位相序与最小绿约束
- 设备编译前校验清单

## 通用护栏
- 所有结论必须引用供给、需求、状态或执行结果中的证据；缺少证据时输出需要补采的数据。
- 优先级顺序：安全底线、防溢流防锁死、基本服务公平、效率、体验。

## 方案生成边界

方案生成负责把控制策略指令转化为信号机可执行或可校核的计划，包括动态子区划分、协调参数、单点配时、特殊场景补丁、下发方式、回滚条件和迭代窗口。不得在缺少策略目标和约束的情况下直接给出秒数。

## 动态子区和协调方案

- 当拥堵表现为连续路段、相邻路口互相影响或面状扩散时，应先判断是否需要动态子区划分。拥堵关联强、排队传导明显、共用瓶颈或受同一事件影响的路口应划入同一控制子区。
- 干线协调方案应明确公共周期、协调相位、相位差、绿波速度、协调方向和退出条件。协调速度应贴近实测平均速度，偏离实际车速时应优先调整相位差或退出强协调。
- 区域协同方案应明确边界路口、核心保护路口、瓶颈节点和外围分流节点，并给出边界放行、核心出口优先和替代路径协同关系。

## 路口执行计划

- 单点方案应至少包含周期、关键相位绿信比、相位相序、最小绿、黄灯、全红、行人清空、放行模式和回滚条件。
- 失衡纠偏场景应以小步长调整绿信比和相序为主，优先削减空放并保证支路最小服务。
- 防溢流场景应优先保护出口清空和下游余量，可采用量出为入、上游限放、关键转向清空和相位重构。
- 慢行保护场景应校验行人过街清空、非机动车等待区、安全岛和右转冲突控制，不得以机动车效率压缩安全时间。
- 特殊时段方案必须设置生效时段、触发条件和快速回切条件，避免学校、医院、商圈等短时策略在平峰继续造成空放。

## 下发与设备适配

- 方案下发前应完成多品牌信号机参数适配，确保周期、相位、阶段、相序、相位差、过渡方式和执行模式可被目标控制机识别。
- 若控制机不支持复杂策略，应裁剪为可执行子策略，优先保留安全清空、基本服务、防溢流和最近验证方案。
- 高风险方案必须采用人工确认、分步下发和平滑过渡。涉及相位重构、右转限行、边界控流、黄灯全红调整和应急策略时，应保留回滚方案。

## 方案校验

- 周期不得突破 `max_cycle_s`，绿灯不得低于最小绿，行人和非机动车清空时间不得低于安全要求。
- 下游阻塞、排队存储不足或连续周期不清空时，不得继续简单增绿，应优先校验出口能力和上游限放。
- 若仿真或在线评价显示延误、排队、溢流风险、冲突风险任一核心指标恶化，应回流方案生成或控制策略阶段复核。
