# 路口评价反馈参考规则

## 文档依据
- 评价报告模板
- 效率、安全、执行稳定性和问题改善指标
- 闭环触发与经验沉淀规则

## 通用护栏
- 所有结论必须引用供给、需求、状态或执行结果中的证据；缺少证据时输出需要补采的数据。
- 优先级顺序：安全底线、防溢流防锁死、基本服务公平、效率、体验。

## 评价目标

评价反馈应围绕效率、安全、公平、体验和执行稳定性形成闭环校验。评价结论必须回答三个问题：是否达到目标、是否引入新的风险、是否需要回流到场景认知、问题诊断、控制策略或方案生成。

## 指标体系

- 效率指标：平均车速提升率、平均延误降低率、排队长度变化、停车次数变化、行程时间可靠性和绿波通过率。
- 安全指标：冲突点减少率、行人和非机动车冲突变化、黄灯全红清空合规性、右转冲突风险和事故或急刹事件变化。
- 配时合理性指标：绿灯利用率、空放率、关键相位饱和度、相位饱和度差异、周期稳定性和协调相位到达匹配度。
- 问题改善指标：拥堵时长缩短率、溢流风险下降、连续周期不清空次数下降、失衡问题减少、外部事件恢复时间缩短。
- 体验与合规指标：投诉量、舆情满意度、公交准点率、行人等待时间、信号机故障率、通信稳定性和人工接管次数。

## 评价窗口

- 短时策略应使用分钟级或评价窗口级指标，关注排队、溢流、冲突和执行稳定性。
- 日常优化应形成日、周、月评价报告，识别反复未达标的路口、干线或区域。
- 特殊事件、大型活动、恶劣天气和学校医院时段应单独建档，避免与普通工作日均值混淆。

## 回流规则

- 若目标未达成但风险未恶化，优先回流到控制策略阶段，重新调整目标优先级、策略包和触发退出规则。
- 若执行计划违反约束、核心指标恶化或新增溢流和安全风险，优先回流到方案生成阶段，收紧配时参数、回滚条件和下发护栏。
- 若连续多轮优化后仍无法改善，应回流到场景认知和问题诊断，复核是否存在静态设施瓶颈、交通秩序问题、外部事件、数据质量问题或知识库未覆盖的新场景。
- 若数据缺失、检测异常或执行状态不可信，应触发降级护栏，评价结果只能作为低置信度参考。

## 经验沉淀

- 每次闭环应沉淀“场景标签 - 问题码 - 策略包 - 方案参数 - 评价结果 - 回滚情况”的结构化记录。
- 对有效策略应记录适用边界和退出条件，对无效策略应记录失败原因和禁用条件。
- 对学校、医院、商圈、景区、枢纽、桥隧、施工和恶劣天气等重复场景，应持续更新日历化方案、阈值和应急预案。
