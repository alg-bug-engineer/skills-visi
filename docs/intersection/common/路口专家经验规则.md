# 路口专家经验规则

本文是交通专家维护路口经验的开发期源稿。工程师运行专家经验编译器后，编译器会把本文件中的经验条目解析为结构化 IR，并生成或更新运行时使用的 `SKILL.md`、`references/rules.md`、YAML 配置和脚本草稿。

## 写作原则

- 每条经验必须有稳定 ID，格式为 `EXP-SCENE-001`、`EXP-DIAG-001` 等；ID 发布后不要复用给其他含义。
- 专家优先写交通业务现象、判别依据、适用边界和误判提醒；字段路径、脚本函数和 PG 表映射由编译器和工程师补齐。
- 涉及阈值数字时，应写清指标、单位和语义；编译器会优先映射到 `thresholds.yaml` 既有键，无法映射时生成建议新增键。
- 经验可以建议生成多个目标，例如 `rules_md`、`diagnosis_checklist`、`thresholds`、`scene_type_rules`、`skill_step`、`script_draft`。
- 脚本逻辑只生成草稿，不直接覆盖核心运行脚本；工程师审核后再迁入正式脚本。

## 条目格式

新增经验请按以下模板填写：

```markdown
### EXP-DIAG-001 经验名称

- 阶段：problem_diagnosis
- 类型：checklist_item | threshold | issue_code | scene_type_rule | skill_step | script_logic | narrative_rule
- 现象：描述交通专家在现场或数据中看到的表现。
- 判别依据：列出可用指标、现场线索或上下文标签。
- 适用边界：说明适用于哪些路口、时段、场景或数据条件。
- 不要误判为：说明容易混淆的问题类型。
- 建议生成：diagnosis_checklist, thresholds, rules_md, script_draft
- 运行时意图：说明希望运行时如何使用该经验。
```

## 场景认知经验

### EXP-SCENE-001 供给-需求-状态三维画像

- 阶段：scene_cognition
- 类型：narrative_rule, skill_step
- 现象：路口场景认知需要先形成事实底座，再进入问题诊断。
- 判别依据：道路设施、车道渠化、信号控制、交通需求、运行状态、数据质量。
- 适用边界：所有路口场景认知任务。
- 不要误判为：最终主因诊断或配时策略生成。
- 建议生成：rules_md, skill_step
- 运行时意图：场景认知输出时先描述供给，再描述需求，最后描述运行状态和质量标签；本阶段不输出问题归因和策略动作。

### EXP-SCENE-002 结构性瓶颈供给线索

- 阶段：scene_cognition
- 类型：narrative_rule
- 现象：三进两出、出口道少于进口道、短车道排队溢出、相邻路口间距过短、转弯半径过小或路口面积过大时，路口供给存在结构性瓶颈线索。
- 判别依据：进口出口车道数、转向车道长度、相邻路口间距、路口面积、渠化与车道功能。
- 适用边界：静态供给画像完整或可由 PG 静态表补采的路口。
- 不要误判为：单纯配时不合理。
- 建议生成：diagnosis_checklist, rules_md, thresholds
- 运行时意图：场景认知应将这些线索写入供给画像和上下文标签；问题诊断再结合动态证据判断是否成为主因。

### EXP-SCENE-003 强吸引点上下文标签

- 阶段：scene_cognition
- 类型：checklist_item, narrative_rule
- 现象：学校、医院、商圈、景区、交通枢纽、桥隧匝道、施工点会改变需求峰值、慢行冲突、停车绕行或排队存储。
- 判别依据：POI 标签、投诉台账、现场调研、施工或活动标签。
- 适用边界：路口周边存在强吸引或强约束对象。
- 不要误判为：普通周期或绿信比分配问题。
- 建议生成：scene_cognition_checklist, rules_md
- 运行时意图：将强吸引点写入 `context_tags`，供诊断阶段判断特殊需求、外部干扰和慢行保护。

### EXP-SCENE-004 状态指标联合判断

- 阶段：scene_cognition
- 类型：threshold, narrative_rule
- 现象：单一饱和度无法完整说明拥堵状态。
- 判别依据：饱和度、平均延误、排队长度、排队存储比、溢流风险、停车次数、绿灯利用率、空放率。
- 适用边界：动态 DWS 时序可用，或任务输入提供关键运行指标。
- 不要误判为：只凭饱和度得出拥堵结论。
- 建议生成：rules_md, thresholds, script_draft
- 运行时意图：状态画像和指标解读应联合多个指标判断压力等级、拥堵阶段和数据质量冲突。

### EXP-SCENE-005 拥堵空间形态与发展阶段

- 阶段：scene_cognition
- 类型：narrative_rule, scene_type_rule
- 现象：排队可能表现为点状、线状或面状，并经历萌芽、扩散、消散阶段。
- 判别依据：单点排队、连续路段排队、绿波断裂、相邻路口传导、延误和溢流风险趋势。
- 适用边界：有路口、路段或相邻路口时序证据时。
- 不要误判为：单点孤立问题。
- 建议生成：rules_md, scene_type_rules
- 运行时意图：将拥堵空间形态和阶段写入 `congestion_profile`，作为诊断优先级和策略范围判断依据。

### EXP-SCENE-006 数据冲突与不确定性

- 阶段：scene_cognition
- 类型：narrative_rule, threshold
- 现象：低饱和度同时出现高延误或长排队时，指标之间存在冲突。
- 判别依据：饱和度较低、平均延误较高、排队较长、下游阻塞或现场干扰标签。
- 适用边界：运行指标存在明显矛盾或关键数据缺失。
- 不要误判为：直接判定为配时不足。
- 建议生成：rules_md, thresholds, script_draft
- 运行时意图：写入 `quality_tags` 或 `uncertainty`，并提示补采下游阻塞、停车占道、公交停靠、慢行冲突或渠化数据。

### EXP-SCENE-007 路口道路等级与供给优先级

- 阶段：scene_cognition
- 类型：narrative_rule, skill_step
- 现象：不同等级道路相交决定路口重要程度和主次通行关系，例如主干路-主干路是城市路网关键节点，主干路/次干路与支路相交时应明确主路优先，支路-支路主要服务局部微循环。
- 判别依据：进口 link 的 `dim_link_info.road_level` 编码、进口方向道路等级组合、主次道路等级差异。
- 适用边界：路口进口 link 能关联 `dim_link_info`，或任务输入提供道路等级信息。
- 不要误判为：仅由当前流量大小决定的动态需求强弱。
- 建议生成：rules_md, skill_step
- 运行时意图：将进口道路等级解析为 `supply_profile.road_levels`、`road_grade_combination` 和 `intersection_importance`，为后续诊断判断主流向优先级与治理范围提供上下文。

### EXP-SCENE-008 路口几何形态与冲突模式

- 阶段：scene_cognition
- 类型：narrative_rule, scene_type_rule
- 现象：十字路口组织最常见，T/Y 型冲突点较少但转弯与视距约束突出，X 型斜交对左转和视距不利，环岛、多路或错位畸形路口冲突模式复杂，行人过街路口需关注慢行保护。
- 判别依据：`dim_inter_info.inter_type`、`inter_proto`、进口方向数量、进口方向夹角或现场形态标签。
- 适用边界：静态基础信息可识别路口类型，或现场调研能补充形态。
- 不要误判为：单纯相位参数不合理；复杂形态应先作为供给约束和冲突背景。
- 建议生成：rules_md, scene_type_rules
- 运行时意图：将路口形态归一为 `supply_profile.intersection_shape`、`leg_count` 和形态相关 `static_flags`，并在场景认知输出中说明可能的交织、冲突和特殊组织约束。

## 问题诊断经验

### EXP-DIAG-002 动态供需失衡

- 阶段：problem_diagnosis
- 类型：checklist_item, issue_code
- 现象：高峰小时流量接近或超过通行能力，节假日、开学日或强吸引点导致需求激增。
- 判别依据：饱和度、交通量、连续周期不清空、主流向占比、特殊时段或特殊吸引点标签。
- 适用边界：动态流量和能力证据可用。
- 不要误判为：静态渠化短板或外部事件主导。
- 建议生成：thresholds, rules_md
- 运行时意图：由 `demand_pressure_perception`、`attractor_demand_pressure`、`cycle_timing` 等检查项覆盖，触发过饱和、需求失衡或特殊需求相关问题，并在成因维度中体现需求贡献。

### EXP-DIAG-003 交通秩序与外部事件干扰

- 阶段：problem_diagnosis
- 类型：checklist_item, issue_code, scene_type_rule
- 现象：违停、双排停车、网约车和快递临停、公交停靠、施工、事故、恶劣天气、大型活动或临时管制会导致通行能力突降或需求突变。
- 判别依据：投诉台账、现场调研、施工事件、事故标签、公交站点、出入口干扰、天气或活动信息。
- 适用边界：外部扰动有上下文证据支撑。
- 不要误判为：单纯信控参数不适配。
- 建议生成：diagnosis_checklist, scene_type_rules, rules_md
- 运行时意图：优先判定为秩序或事件主导，降低信控优化上限，输出协同治理或应急策略建议。

### EXP-DIAG-004 路口服务失衡

- 阶段：problem_diagnosis
- 类型：checklist_item, threshold, issue_code
- 现象：各进口、流向或相位饱和度差异过大，部分方向持续排队或高饱和，其他方向空闲。
- 判别依据：失衡指数、相位饱和度差、车道利用率离散、关键相位排队不清空。
- 适用边界：分进口、分流向或分相位运行数据可用。
- 不要误判为：下游阻塞或静态渠化短板单独主导。
- 建议生成：thresholds, rules_md
- 运行时意图：由 `service_imbalance`、`lane_flow_mismatch` 等检查项覆盖，触发服务失衡问题，提升控制参数不适配成因贡献。

### EXP-DIAG-006 绿灯损失或空放

- 阶段：problem_diagnosis
- 类型：checklist_item, threshold, issue_code
- 现象：某相位绿灯期间无车或少车通过，平峰或夜间沿用高峰方案，造成无效等待。
- 判别依据：空放率、绿灯利用率、启动损失、时段方案数、到达流量。
- 适用边界：绿灯利用率或相位放行观测可用。
- 不要误判为：因检测器缺测造成的假空放。
- 建议生成：thresholds, rules_md
- 运行时意图：由 `empty_green`、`plan_granularity` 等检查项覆盖，触发空放或方案精细度问题，并保留数据质量不确定性。

### EXP-DIAG-007 相位相序与慢行保护

- 阶段：problem_diagnosis
- 类型：checklist_item, issue_code, narrative_rule
- 现象：高冲突流向同相放行、左转专用车道无保护相位、全红清空与实际冲突不匹配、学校医院商圈慢行需求高。
- 判别依据：相位相序、信号原子与车道映射、冲突标签、行人非机动车需求、投诉或现场调研。
- 适用边界：冲突风险或慢行需求证据可用。
- 不要误判为：单纯效率问题；涉及安全时安全优先。
- 建议生成：diagnosis_checklist, rules_md, script_draft
- 运行时意图：触发相位冲突或慢行保护不足问题，并提高安全类问题优先级。

### EXP-DIAG-009 诊断优先级

- 阶段：problem_diagnosis
- 类型：narrative_rule, scene_type_rule
- 现象：多个问题同时出现时，需要先处理安全和扩散风险，再处理效率和体验。
- 判别依据：冲突风险、溢流锁死、下游阻塞、过饱和、服务失衡、长排队、空放、方案精细度、投诉和专项保障。
- 适用边界：诊断阶段已形成多个候选问题。
- 不要误判为：按单个分值机械排序。
- 建议生成：rules_md, scene_type_rules
- 运行时意图：优先级排序遵循安全、扩散、供需、效率、结构、体验的顺序。

### EXP-DIAG-010 交通治理感知成因研判

- 阶段：problem_diagnosis
- 类型：checklist_item, threshold, issue_code, narrative_rule
- 现象：高饱和持续、低速运行、高峰出行集中、公交线路和车辆靠站集聚、路内停车占道、强吸引点到达需求增长、进口/转向/出口能力不匹配，会从需求、供给、秩序和外部事件侧共同造成路口运行恶化。
- 判别依据：高饱和持续时间、平均速度、高峰出行系数、强吸引点到达量/通行能力比、公交线路数、单位时间公交集聚数、公交停靠速度差、路内停车数量与主线流量比、进口能力比、流向 V/C、出口汇入车道数与出口道差值、绕行时间。
- 适用边界：图片模板中的阈值只作为诊断口径，必须由场景画像、现场调研、GIS/视频/公交/停车数据或投诉事件证据回链；字段缺失时输出 `data_source_gaps`，不得臆造数值。
- 不要误判为：单纯周期或绿信比分配问题；公交、停车、出入口和出口能力短板主导时信控只做缓解。
- 建议生成：diagnosis_checklist, thresholds, rules_md, checklist_rules_md, script_draft
- 运行时意图：把交通治理感知表转化为可运行检查项，触发 `oversaturation`、`excess_delay`、`external_disturbance`、`downstream_blockage`、`lane_mismatch` 等既有问题码，并输出分流、错峰、公交站点优化、停车秩序、车道功能/可变车道和出口扩容等非信控治理建议。

### EXP-DIAG-011 相位相序与渠化匹配

- 阶段：problem_diagnosis
- 类型：checklist_item, issue_code, script_logic
- 现象：专用导向车道无保护相位、阶段内对向直左/相交直直冲突、专用车道无对应转向放行。
- 判别依据：`supply_profile.channelization`、`control_profile.stage_detail[].release_movements`、相位数；按信号阶段 + 逐进口方向核查，规则 A–D 详见 `专家经验调试/路口交通问题诊断方法.md` §1。
- 适用边界：渠化 + 相位/阶段数据齐全时可精细判定；无 `stage_detail` 且无 `phase_sequence` 时标记数据不足。
- 不要误判为：右转专用道相位约束问题（右转不纳入相位约束判定）。
- 建议生成：diagnosis_checklist, thresholds, rules_md, checklist_rules_md, script_draft
- 运行时意图：由 `diagnosis_static_logic.analyze_phase_channel_match` 与 `check_static_constraints` 产出 `phase_channel_mismatch` 问题码及 `phaseChannelMatch` 报告行。

### EXP-DIAG-012 渠化设计与车流量匹配

- 阶段：problem_diagnosis
- 类型：checklist_item, threshold, issue_code, script_logic
- 现象：左转/直行需求与车道功能配置错位、同进口转向饱和度失衡、结构偏差指数过大。
- 判别依据：分进口转向流量占比、车道功能配置比、分转向饱和度；规则 A–D 见 `专家经验调试/路口交通问题诊断方法.md` §2。
- 适用边界：优先高峰时段转向流量；某进口无渠化或无转向流量时跳过，全部缺数据则标记 `no_data`。
- 不要误判为：单纯信控绿信比不足；结构错位需渠化或可变车道治理。
- 建议生成：diagnosis_checklist, thresholds, rules_md, checklist_rules_md, script_draft
- 运行时意图：由 `diagnosis_static_logic.analyze_lane_flow_match` 驱动 `lane_flow_mismatch` 检查项，触发 `lane_mismatch` 问题码。

### EXP-DIAG-013 进口出口漏斗效应

- 阶段：problem_diagnosis
- 类型：checklist_item, issue_code, script_logic
- 现象：进口直行车道数大于对向出口车道、进口方向数多于出口，导致路口内交织与排队回堵。
- 判别依据：分方向直行车道数、对向出口车道总数；规则 A–B 见 `专家经验调试/路口交通问题诊断方法.md` §3。
- 适用边界：左转/右转/左直混行不参与直行动线漏斗比对；无车道明细时回退场景认知预扫描标记。
- 不要误判为：总进口车道与总出口车道简单比较（须按直行断面成对排查）。
- 建议生成：diagnosis_checklist, rules_md, checklist_rules_md, script_draft
- 运行时意图：由 `diagnosis_static_logic.analyze_funnel_effect` 驱动 `funnel_effect` 检查项，输出 `inOutLanes` 对齐证据。
