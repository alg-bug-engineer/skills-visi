# 路口静态问题诊断方法（精简版）

> 阈值以 `thresholds.yaml` 为准；运行时由 `diagnosis_static_logic.py` 实现。  
> 路口级结论：**任一触发规则命中 → 存在问题**；全部通过 → 无问题；关键字段缺失 → 数据不足。

---

## 1、相位相序与渠化是否匹配

**数据**：`supply_profile.channelization`；`control_profile.stage_detail[].release_movements`（无阶段明细时回退 `phase_sequence`）。右转车流不纳入相位约束，右转专用道仍计入进口车道数。

**粒度**：按信号阶段 + 逐进口方向核查；任一命中 → 不匹配。

| 规则 | 触发条件 |
| --- | --- |
| A | 阶段内对向一方放直行、另一方放左转，且任一方进口车道数 ≥ 3 |
| B | 阶段内两条垂直进口同时放直行 |
| C | 有左转专用道但环图中无对应左转专用位；或左转仅与对向直行同放且该进口车道数 ≥ 3 |
| D | 有专用直行车道/左转车道，但全方案无对应转向放行 |

**数据不足**：无渠化；或既无 `stage_detail` 又无 `phase_sequence`。


---

## 2、渠化设计与车流量是否匹配

**数据**：`channelization`、分进口转向流量（`demand_profile.movement_volume`）、分转向饱和度（`traffic_state.movement_saturation`）；阈值见 `thresholds.yaml` 中 `channelization.*`、`static.lane_utilization_cv`、`static.lane_mismatch_index`。

**粒度**：按进口方向逐一判定；任一进口或路口级规则命中 → 不匹配。优先取高峰时段流量。

**预处理**：逐进口汇总车道配置比（左:直:右）与流量占比；混行车道按可服务转向均分。

| 规则 | 触发条件 |
| --- | --- |
| A | 左转流量占比 ≥ 25% 且无左转专用道（含仅混行） |
| B | 直行流量占比 ≥ 60% 且有效直行车道 ≤ 1 且直行饱和度 ≥ 0.85 |
| C | 结构偏差指数（\|配置比 − 流量比\| 之和）≥ 0.50 |
| D | 同进口某转向饱和度 ≥ 0.90 且另一转向 ≤ 0.50 |


**数据不足**：某进口无渠化或无转向流量 → 跳过该进口；全部进口均缺数据 → 无法判定。

---

## 3、进出口是否存在漏斗效应

**数据**：`supply_profile.channelization` / `lanes`；回退 `static_flags`、`funnel_details`。无车道明细时用 link 级 `lane_num`、`lane_info`、`turn_move`。

**判定对象**：进口**直行动线**在对向/同向出口能否承接。左转、右转、左直混行不计入直行断面；**勿简单比较进出口总车道数**。

**粒度**：按进口方向 × 对向出口成对排查；任一对命中 → 存在漏斗效应。

| 规则 | 触发条件 |
| --- | --- |
| A | 进口直行车道数 > 对向出口车道总数（如东 3 直 → 西 2 出） |
| B | 有进口的方向数 > 有出口的方向数 |

**数据不足**：无进出口 link 数据且无预扫描标记。
